## 一、操作系统的基本概念

### 1.1 定义
**OS** = 控制和管理计算机系统软/硬件资源、合理调度任务与分配资源，并为用户及其他软件提供统一接口与运行环境的<mark style="background: #d4e3ff">**最基本的系统软件**</mark>。

### 1.2 五大功能

| 功能         | 核心职责                      |
| ---------- | ------------------------- |
| **处理机管理**  | 进程控制、进程同步、进程通信、死锁处理、处理机调度 |
| **存储器管理**  | 内存分配回收、地址映射、内存保护共享、内存扩充   |
| **文件管理**   | 文件存储空间管理、目录管理、文件读写、访问保护   |
| **设备管理**   | 缓冲管理、设备分配、设备驱动、虚拟设备       |
| **用户接口管理** | 使用户更方便、更可靠地操作硬件设备和运行用户程序  |

> 🔑 <mark style="background: #d4e3ff">库函数 ≠ 操作系统接口，系统调用才是</mark>。库函数是对系统调用的封装。

### 1.3 四大特征

```
并发 ←→ <mark style="background: #d4e3ff">共享（互为前提，最基本）</mark>
  ↓
<mark style="background: #d4e3ff">虚拟（时分复用 + 空分复用）</mark>
  ↓
<mark style="background: #d4e3ff">异步（进程走走停停，不可预知）</mark>
```

| 特征 | 关键词 | 易错点 |
|------|--------|--------|
| **并发** | 同一<mark style="background: #d4e3ff">**时间间隔**</mark>内 | <mark style="background: #d4e3ff">≠ 并行（同一**时刻**）</mark> |
| **共享** | 互斥共享 / 同时访问 | 临界资源必须互斥 |
| **虚拟** | <mark style="background: #d4e3ff">一个物理实体 → 多个逻辑对应物</mark> | 时分复用（CPU）+ 空分复用（虚存） |
| **异步** | 进程以不可预知速度推进 | 靠同步机制保证结果正确 |

## 二、操作系统发展历程

```
手工操作 → 单道批处理 → 多道批处理 → 分时 OS → 实时 OS
                                              ↘ 网络 OS / 分布式 OS
```

| 阶段        | 核心特点         | 关键考点                                                          |
| --------- | ------------ | ------------------------------------------------------------- |
| 手工操作      | CPU 长时间等人工操作 | 人机速度矛盾                                                        |
| 单道批处理     | 内存仅一道程序，自动连续 | 自动性、顺序性、单道性                                                   |
| **多道批处理** | 内存多道程序交替执行   | 宏观并行、微观串行；<mark style="background: #d4e3ff">**依赖中断技术**</mark> |
| 分时 OS     | 时间片轮转，人机交互   | <mark style="background: #d4e3ff">同时性、交互性、独立性、及时性</mark>      |
| 实时 OS     | 严格时限内响应      | 硬实时（导弹）vs 软实时（订票）                                             |

## 三、程序运行环境 ★★★

### 3.1 双重工作模式

```
┌─────────────┐
│ 用户态（目态）│ ← 只能执行非特权指令
├─────────────┤
│ 内核态（管态）│ ← 可执行特权指令（关中断、I/O、置时钟…）
└─────────────┘
```

> <mark style="background: #d4e3ff">**进入内核态的唯一合法途径：中断或异常**</mark>

### 3.2 中断和异常 ★★★

```
        中断/异常
       ┌───┴───┐
     外中断    异常（内中断）
  （CPU 外部） （CPU 内部）
       │         │
  ┌────┴────┐  ┌──┼──┐
可屏蔽  不可屏蔽 故障 自陷 终止
(INTR)  (NMI)  (Fault)(Trap)(Abort)
```

| 类型 | 触发方式 | 处理后返回 | 示例 |
|------|---------|-----------|------|
| <mark style="background: #d4e3ff">**故障**</mark> | 指令执行出错 | <mark style="background: #d4e3ff">**原指令重执行**</mark> | 缺页、除零、非法操作码 |
| <mark style="background: #d4e3ff">**自陷**</mark> | 程序主动触发 | <mark style="background: #d4e3ff">下一条指令</mark> | 系统调用（Trap 指令） |
| <mark style="background: #d4e3ff">**终止**</mark> | 硬件致命故障 | <mark style="background: #d4e3ff">不可恢复，终止程序</mark> | 控制器错、校验错 |

> 🔑 <mark style="background: #d4e3ff">软件中断 = 故障 + 自陷；硬件中断 = 终止 + 外中断</mark>

### 3.3 系统调用 ★★★

<mark style="background: #d4e3ff">**处理过程**：传参 → 陷入指令（用户态→内核态）→ 查系统调用表 → 执行服务 → 返回用户态</mark>

| 分类 | 典型调用 |
|------|---------|
| 设备管理 | read/write |
| 文件管理 | open/close/create/delete |
| 进程控制 | fork/exec/wait/exit |
| 进程通信 | pipe/shmget/msgget |
| 内存管理 | brk/mmap |

## 四、操作系统结构 ★★

```
架构演进：分层 → 模块化 → 宏内核 → 微内核 → 外核
```

| 架构 | 模块位置 | 优势 | 劣势 |
|------|---------|------|------|
| **分层** | 层层单向依赖 | 易调试验证 | 层次难划分、效率低 |
| **模块化** | 内核态，自由调用 | 灵活、可并行开发 | 模块依赖乱，调试难 |
| <mark style="background: #d4e3ff">**宏内核**</mark> | 全部在内核态 | <mark style="background: #d4e3ff">**性能高**</mark> | 代码臃肿，一个模块崩全系统崩 |
| <mark style="background: #d4e3ff">**微内核**</mark> | 仅 IPC/调度/中断在内核 | <mark style="background: #d4e3ff">**可靠、安全、可扩展**</mark> | <mark style="background: #d4e3ff">**效率低**（频繁切换）</mark> |
| <mark style="background: #d4e3ff">**外核**</mark> | <mark style="background: #d4e3ff">只做资源分配+隔离</mark> | 极简、灵活、高性能 | 库 OS 开发复杂 |

> 🔑 <mark style="background: #d4e3ff">微内核"添加服务不动内核""服务崩不拖垮系统"——常设陷阱</mark>
> 🔑 <mark style="background: #d4e3ff">宏内核"效率高" vs 微内核"可靠性高"——常互为干扰项</mark>

## 五、操作系统引导

```
通电 → BIOS (ROM) → POST 自检 → 选启动设备
    → 加载 MBR (硬盘首扇区) → 解析分区表 → 定位活动分区
    → 加载 PBR → 加载启动管理器 (GRUB/Bootmgr)
    → 加载 OS 内核 → 内核初始化 → 启动 init 进程
```

| 关键考点 | 位置 |
|---------|------|
| <mark style="background: #d4e3ff">BIOS 自举程序</mark> | <mark style="background: #d4e3ff">**ROM**</mark> |
| <mark style="background: #d4e3ff">操作系统引导程序（启动管理器）</mark> | <mark style="background: #d4e3ff">**硬盘**活动分区引导扇区</mark> |
| <mark style="background: #d4e3ff">OS 内核最终加载到</mark> | <mark style="background: #d4e3ff">**RAM**</mark> |
| <mark style="background: #d4e3ff">初始化时创建</mark> | <mark style="background: #d4e3ff">**中断向量表**</mark> |

## 六、虚拟机

### VMM 两种架构

| | 第一类（裸金属） | 第二类（寄居） |
|------|---------------|-------------|
| 运行位置 | <mark style="background: #d4e3ff">**直接跑在硬件上**</mark> | 跑在 Host OS 上（用户态进程） |
| 特权级 | <mark style="background: #d4e3ff">最高</mark> | 与普通应用同级 |
| 性能 | <mark style="background: #d4e3ff">高</mark> | 较低 |
| 代表 | VMware ESXi、KVM、Xen | VMware Workstation、VirtualBox |

> 🔑 <mark style="background: #d4e3ff">第一类：无 Host OS，VMM 替代 OS 管理硬件</mark>
> 🔑 <mark style="background: #d4e3ff">VMM 代码量 ≪ 完整 OS 代码量</mark>

## 七、高频易错点汇总

1. <mark style="background: #d4e3ff">**并发 ≠ 并行**：并发 = 同一时间间隔交替；并行 = 同一时刻真正同时</mark>
2. <mark style="background: #d4e3ff">**系统调用接口 ≠ 库函数**：库函数是封装，系统调用才是真正接口</mark>
3. <mark style="background: #d4e3ff">**中断处理程序一定是 OS 程序**，不是用户程序</mark>
4. <mark style="background: #d4e3ff">**Trap 指令在用户态执行**，但触发后 CPU 切到内核态</mark>
5. <mark style="background: #d4e3ff">**关中断是特权指令**，只能在核心态执行</mark>
6. <mark style="background: #d4e3ff">**缺页属于异常（内中断）**，不属于外中断</mark>
7. <mark style="background: #d4e3ff">**微内核更可靠但效率更低**，宏内核反之</mark>
8. <mark style="background: #d4e3ff">**模块化 OS 一个模块崩→整个内核崩**，"容错"是微内核的特点</mark>
9. <mark style="background: #d4e3ff">**外核不做进程调度**，调度由库 OS 完成</mark>
10. <mark style="background: #d4e3ff">**中断向量表**在 OS 初始化创建；硬盘分区表/根目录/索引节点在磁盘分区/格式化时创建</mark>
