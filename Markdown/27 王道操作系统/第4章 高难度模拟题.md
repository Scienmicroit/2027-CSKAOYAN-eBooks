## 答案与解析

---

**01. A**

- **A ✅** 两种链接分配都支持顺序访问（从首块开始顺序读取所有块）。隐式链接不支持随机访问（指针在数据块中，需顺序读取）；显式链接的 FAT 在内存中，查表后可直接定位任意块，因此支持随机访问。
- B ❌ 隐式链接的指针存储在**磁盘的数据块**中，不在内存中。读取第 i 块意味着执行 i 次磁盘读操作（读第 1 块提取指向第 2 块的指针，读第 2 块提取指向第 3 块的指针……），时间开销与 i 成正比且均为磁盘 I/O 延迟。
- C ❌ FAT 表项中每一行确实记录下一盘块号，但文件的**起始块号**存储在文件目录项（FCB/inode）中，不在 FAT 本身存储。起始块号需要另有位置记录。
- D ❌ 显式链接的文件目录项中只需记录起始块号，后续块号由 FAT 链式索引。隐式链接的文件目录项中通常需记录**起始块号和结束块号**（或文件长度），以支持遍历到文件末尾时判断终止条件。

---

**02. C（3 次）**

精确计算各地址范围的字节边界：

- 盘块大小 = 4096 B
- 直接地址（10 个）：覆盖字节范围 0 ~ 10×4096-1 = **0 ~ 40,959**
- 一次间接：1024 × 4096 = 4,194,304 B
  一次间接覆盖范围：40,960 ~ 40,960 + 4,194,304 - 1 = **40,960 ~ 4,235,263**
- 二次间接：1024² × 4096 = 4,294,967,296 B（4GB）
  二次间接覆盖范围：4,235,264 ~ 4,235,264 + 4,294,967,296 - 1 = **4,235,264 ~ 4,299,202,559**

偏移量 4,235,264 ≥ 4,235,264，因此不在一次间接范围内，而落在**二次间接的起始位置**（二次间接的第一个字节）。

访盘次数 = 1（读主索引块）+ 1（读二级索引块）+ 1（读数据块）= **3 次**。

- A ❌ 1 次仅对应直接地址。
- B ❌ 误认为 4,235,264 仍在一次间接范围内（未精确计算边界）。
- **C ✅** 正确。
- D ❌ 4 次对应三次间接。

**【陷阱分析】** 4,235,264 是二/三次间接边界处的精确值：一次间接末字节 = 4,235,263（= 10×4096 + 1024×4096 - 1），4,235,264 正好是二次间接的起始字节。需精确计算边界，否则极易误判为一次间接（2 次访问）。

---

**03. D**

- A ✅ 正确。进程打开文件表（每进程的文件描述符表）是私有的，记录 fd 到系统打开文件表项的映射。
- B ✅ 正确。系统打开文件表是全局的，包含打开计数器。
- C ✅ 正确。同一进程两次 open 同一文件，获得两个不同的 fd，在系统打开文件表中创建两个独立的表项（各有独立文件位置指针），指向同一个 inode。
- **D ❌** 文件描述符（fd）是**进程打开文件表**的索引号，不是**系统打开文件表**的索引号。进程通过 fd 在进程自己的表中找到对应项，此项中存储了指向系统打开文件表项的指针。这是两级表结构的核心设计。

---

**04. A**

逐步追踪 inode 链接计数变化：

| 步骤 | 操作 | 计数变化 | 计数值 | 说明 |
|:----:|:----:|:--------:|:------:|:----:|
| 初始 | — | — | 1 | 文件 F 本身占用 1 个目录项 |
| ① | 创建硬链接 G | +1 | **2** | 新增目录项指向同一 inode |
| ② | 创建硬链接 H | +1 | **3** | 再增一个目录项 |
| ③ | 删除文件 F | -1 | **2** | 删除 F 的目录项，仅减计数。因 count > 0，inode 和数据保留 |
| ④ | 删除文件 G | -1 | **1** | 再减一，H 仍指向该 inode，数据完整保留 |

因此序列为：① 2 → ② 3 → ③ 2 → ④ 1，选项 **A** 正确。

- B ❌ 忽略第二次硬链接的计数增加。
- C ❌ 删除 F 后计数未减（误认为删除"原文件"会保留计数）。
- D ❌ 最后一步计数归零错误。

**【核心考点】** 硬链接之间完全对等，不存在"原文件"的特殊地位。删除任何一个链接都只是将 count 减 1。

---

**05. C（104）**

行号 i 和列号 j 均从 1 开始，盘块号从 1 开始。

标准公式为：**盘块号 $= n(i-1) + j$**，其中 n 为字长。

代入 n = 32, i = 4, j = 8：
盘块号 = 32 × (4-1) + 8 = 96 + 8 = **104**。

验证：第 1 行覆盖盘块 1~32，第 2 行覆盖 33~64，第 3 行覆盖 65~96，第 4 行第 1 列 = 97, 第 2 列 = 98, …, 第 8 列 = 104。✅

- A ❌ 96 = 32×(4-1) = 仅考虑行偏移，未加列号。对应第 3 行最后一列（第 32 列）。
- B ❌ 103 = 32×(4-1)+(8-1) = 误将列号按 0 基计算。如盘块号从 0 开始则正确（103 = 0-based），但题目明确盘块号从 1 开始。
- **C ✅** 正确。
- D ❌ 111 = 32×4-8+... 非标准公式。

---

**06. D**

- A ✅ 成组链接法将空闲盘块分组，每组的第一块记录下一组块号，超级块中的空闲栈管理第一组。
- B ✅ 栈不空时直接从栈中取出，仅修改内存中的超级块数据，无需磁盘 I/O。
- C ✅ 栈满时，将当前栈内容写入回收的盘块，然后以该回收盘块为起点建立新栈。
- **D ❌** 回收空闲块时，若栈未满（s_nfree < 100），则直接将盘块号压入栈，只需更新内存中的超级块，**无需触发磁盘写操作**。只有当栈满时才需要写盘。选项说"每次回收都一定触发磁盘写操作"是错误的。

---

**07. C**

- A ❌ 超级块对象除类型标识和总块数外，还包含块大小、空闲块数、挂载信息、文件系统操作函数指针等多方面信息。
- B ❌ VFS inode 对象（vnode）存储**元数据**（权限、大小、时间戳等），但**不存储实际文件数据内容**。实际数据存储在磁盘数据块中。
- **C ✅** dentry 对象缓存一个路径分量到 inode 的映射关系，不存储完整路径。多个路径分量通过 dentry 树串联表示完整路径。
- D ❌ 文件对象与打开文件描述相关联，但并非与文件描述符一一对应。通过 dup() 或 fork() 可使多个 fd 指向同一个文件对象（共享文件位置）。

---

**08. D**

- A ✅ 根文件系统必须先挂载，其他文件系统挂载到以根为起点的目录树上。
- B ✅ 挂载点必须是已存在的目录，挂载后该目录的原内容被隐藏。
- C ✅ 同一文件系统可挂载到多个挂载点，每个挂载点看到的内容相同。
- **D ❌** 挂载**不改变**被挂载文件系统根目录的 inode 编号。根目录在文件系统格式化时已分配固定 inode 编号（如 ext 系列通常为 2），挂载不会将其改为 0。inode 编号 0 通常保留作为"空/无效"标记。

---

**09. A（仅 I、II、III）**

- **I ✅** 磁盘 inode 中通过直接/间接地址项存放文件数据块的盘块号，这是定位文件数据的基础。
- **II ✅** 文件打开时，磁盘 inode 被读入内存缓冲区的 inode 表中；关闭文件时（或定期同步），修改后的 inode 写回磁盘。
- **III ✅** 内存 inode 包含磁盘 inode 没有的字段：i_count（访问计数，决定何时可释放）、i_lock（锁定状态，防止并发修改）、i_dev（逻辑设备号）、i_hash（散列链指针）等。
- **IV ❌** 磁盘 inode 为了节省外存空间，格式紧凑；内存 inode 增加了管理字段，因此**内存 inode 通常大于磁盘 inode**。两者大小不一致。

因此仅 I、II、III 正确，选项 **A** 正确。

---

**10. D**

- A ✅ 平均查找次数 = 索引表顺序查找次数 + 组内顺序查找次数 = $m/2 + (N/m)/2 = m/2 + N/(2m)$。公式正确。
- B ✅ 令 $f(m) = m/2 + N/(2m)$，求导得 $f'(m) = 1/2 - N/(2m²) = 0$，解得 $m = \sqrt{N}$。最优。
- C ✅ 代入 N=10000, m=100：$100/2 + 10000/200 = 50 + 50 = 100$。正确。
- **D ❌** 索引顺序文件在某些条件下可能**劣于**顺序文件。例如当 $N = 2$ 时，顺序文件平均比较 1 次；索引顺序文件若分 2 组（每组 1 条），索引表比较 1 次 + 组内比较 0.5 次 = 1.5 次 > 1 次。此外对于极小的 N，索引表的额外开销导致查找次数反而更多。该选项表述绝对化，错误。

---

**11. D（全部正确）**

- **I ✅** 口令保护的方式：创建文件时设定口令，系统存储在 FCB 中。用户访问时需提供口令，系统验证。
- **II ✅** 加密保护通过对文件内容加密，密钥由用户管理。即使存储介质被盗，没有密钥也无法读取文件内容，安全性更高。
- **III ✅** UNIX 权限控制：9 位 = 3 类用户（owner/group/others）× 3 种权限（r/w/x）。正确。
- **IV ✅** ACL 可为每个用户单独设置细粒度权限，灵活性远高于 9 位模型，但每个文件需维护 ACL 列表，存储开销更大。正确。

**12. D（4 项）**

- **I ✅** 单级目录结构中所有文件在同一张目录表，命名必须唯一。创建前需检查是否重名。
- **II ✅** 两级目录：MFD 记录用户名→UFD 指针，UFD 记录用户文件。不同用户 UFD 独立，允许同名文件。
- **III ✅** 无环图目录结构中，每增加一个共享链接，引用计数 +1；删除链接时 -1。计数归零时才真正释放 inode 和数据块。
- **IV ✅** 目录文件是特殊类型的文件，其内容由目录项组成，每个目录项对应一个文件或子目录。这正是目录文件的定义。

全部 4 项均正确。✅

**13. D（4 项）**

- **I ✅** 连续分配的 FCB 需记录起始盘块号和块数（或文件长度），才能通过偏移计算直接定位任意块。
- **II ✅** 隐式链接分配中，目录项记录**起始块号**（指向首块）和**结束块号**（指向尾块），分别用于开始遍历和判断遍历结束。
- **III ✅** 显式链接（FAT）中，每个 FAT 表项对应一个磁盘块，用特殊值区分：-1（文件最后一盘块）、-2（空闲块）。分配时查找值为 -2 的表项，FAT 本身兼具空闲空间管理功能，无需额外结构。
- **IV ✅** 单级索引分配中，每个文件至少分配一个完整的索引块，即使文件只占 1 个数据块，也需一个索引块来存储该数据块的盘块号。这正是索引分配的主要空间开销来源。

全部 4 项均正确。✅

---

**14. C（16MB）**

- 磁盘总容量 = 32GB = 32 × 2³⁰ B = 2³⁵ B
- 盘块大小 = 4KB = 2¹² B
- 总盘块数 = 2³⁵ / 2¹² = 2²³ = 8,388,608 个盘块
- FAT 每个表项占 2B
- FAT 总大小 = 8,388,608 × 2 = 16,777,216 B = **16 MB**

- A ❌ 4MB 可能误将表项算为 1B 或块数算错。
- B ❌ 8MB 可能误将磁盘当作 16GB。
- **C ✅** 正确。
- D ❌ 32MB 可能误将表项算为 4B 或将磁盘当作 64GB。

**15. B（仅 IV）**

【逐一分析】

- **I ✅** 目录项变小后同样盘块可存放更多目录项，检索时读盘次数减少。正确。
- **II ✅** 连续分配在中间插入记录需移动大量后续数据，磁盘 I/O 次数多；隐式链接只需修改相邻块的指针。因此连续分配效率低于隐式链接。正确。
- **III ✅** inode 编号在文件系统格式化时分配，仅在本文件系统内唯一。硬链接通过 inode 编号索引，跨系统时编号冲突，故不能跨文件系统。正确。
- **IV ❌** 打开计数器减至 0 只意味着**系统打开文件表中的表项可被释放**（即从内存中移除该打开记录）。但磁盘上的 inode 是永久性元数据，不会因此被删除。磁盘 inode 的删除（回收）发生在硬链接计数（link count）减至 0 且没有进程以打开方式引用时。**打开计数器 ≠ 硬链接计数**，这是两个完全不同且独立的概念。

因此仅 IV 错误，选项 **B** 正确。
