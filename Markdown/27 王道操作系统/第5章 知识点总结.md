## 一、I/O控制方式（四种）★★★

| 特性 | 程序直接控制 | 中断驱动 | DMA | 通道 |
|------|:--:|:--:|:--:|:--:|
| 传送单位 | 字/字节 | 字/字节 | 数据块 | 多组数据块 |
| 数据路径 | CPU↔设备 | CPU↔设备 | 设备↔内存 | 设备↔内存 |
| CPU干预 | 极高（轮询） | 高（每字中断） | 低（首尾各一次） | 极低（仅发指令） |
| 并行性 | 无 | 部分 | 高 | 最高 |

> 类比：程序直接=反复催裁缝；中断=裁缝做好一件通知一件；DMA=雇秘书处理100件才汇报；通道=秘书自主决策

### DMA 控制器四寄存器：CR、MAR、DR、DC

### DMA 流程：CPU初始化DMA → DMA接管总线成块传输 → 完成中断 → CPU处理结束

## 二、I/O软件层次结构

```
用户层软件（printf/scanf 库函数）
    ↓ 系统调用
设备独立性软件（逻辑名→物理名、缓冲管理、差错控制）
    ↓
设备驱动程序（将抽象请求转为设备指令）
    ↓
中断处理程序（保护现场、处理数据、唤醒进程）
    ↓
硬件（I/O 设备）
```

## 三、缓冲区管理 ★★

### 处理时间公式

| 类型 | 处理每块时间 |
|------|------------|
| **单缓冲** | Max(T, C) + M |
| **双缓冲** | Max(T, C+M) |

> T=设备送缓冲区时间，M=缓冲区→用户区，C=CPU处理时间

### 单 vs 双缓冲判断

| 条件 | 单缓冲 | 双缓冲 |
|------|--------|--------|
| T > C | T+M（CPU等设备） | T（设备连续） |
| T < C | C+M（设备等CPU） | C+M（CPU连续） |

### 循环缓冲

- in 指针 → 空缓冲区，out 指针 → 满缓冲区
- **in追上out** → 输入快于处理 → 输入阻塞
- **out追上in** → 处理快于输入 → 计算阻塞

### Cache vs Buffer

| | Cache | Buffer |
|------|------|------|
| 数据 | 已存在数据的副本，可长期驻留 | 传输中的数据，用完释放 |
| 目的 | 提高重复访问效率 | 提高传输效率与并行性 |

## 四、SPOOLing 技术 ★★

**四个组成部分：** 输入/输出井（磁盘）+ 输入/输出缓冲区（内存）+ 输入/输出进程 + 井管理程序

**核心价值：独占设备 → 共享虚拟设备**

| 特点 | 说明 |
|------|------|
| 提高 I/O 速度 | 低速设备操作 → 磁盘存取 |
| 独占变共享 | 物理设备未分配给任何进程 |
| 以空间换时间 | 磁盘空间换 CPU 等待 |
| 不需要外围机 | 由软件（进程）模拟 |

---

## 五、磁盘调度算法 ★★★

### 存取时间 = 寻道时间 + 旋转延迟 + 数据传输

> 🔑 寻道时间占比最大，调度目标是减少**平均寻道时间**

### 算法对比

| 算法 | 策略 | 优点 | 缺点 |
|------|------|------|------|
| **FCFS** | 按请求顺序 | 公平简单 | 寻道距离大 |
| **SSTF** | 最近优先 | 性能好 | **可能饥饿** |
| **SCAN（电梯）** | 单向到底再折返 | 避免饥饿 | 两端不公 |
| **C-SCAN** | 单向到底，回扫不服务 | 更公平 | 回扫开销 |
| **LOOK/C-LOOK** | 到最远请求就折返 | 实际系统的SCAN | — |

> 🔑 SSD 不需要磁盘调度算法，FCFS 即可（无寻道时间、旋转延迟）

### 减少延迟：扇区交替编号、盘面错位命名

### 提高 I/O：磁盘缓存、提前读、延迟写、虚拟盘(RAM盘)、RAID

## 六、设备分配

**分配顺序：** SDT → DCT → COCT → CHCT（全部成功才算完成）

**逻辑设备名 → 物理设备名：** 通过逻辑设备表 LUT 映射

## 七、SSD vs 机械磁盘（2025考点）

| | 机械磁盘 | SSD |
|------|:--:|:--:|
| 随机访问 | 慢（需寻道） | 极快 |
| 调度算法 | 需要 | **不需要**（FCFS即可） |
| 磨损均衡 | 不需要 | **需要** |
| 读写单位 | 扇区 | 页（读/写）、块（擦除） |

---

## 八、高频易错点

1. **可并行**：处理机与设备、处理机与通道、设备与设备均可并行（单CPU下进程间不行）
2. **共享设备** ≠ 同时访问，是"一段时间内"分时交替
3. **SPOOLing 不需要外围计算机**，由软件进程模拟
4. **通道(Channel)是硬件**，SPOOLing/缓冲池/覆盖是软件
5. **SSD 随机写比随机读慢**（擦除开销大一个数量级）
6. **磁盘地址** = 柱面号·盘面号·扇区号；存满扇区→磁道→柱面
7. **中断处理程序必须快速非阻塞**，运行在中断上下文
8. 每类设备只需一个驱动程序，同类多设备共享
