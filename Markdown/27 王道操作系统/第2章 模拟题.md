**15.** 下列关于死锁处理策略的叙述中，错误的是（　）。

- C. 安全状态一定不会发生死锁，不安全状态一定会发生死锁

## 答案与解析

---

**01. D**

- **I ✅** 进程是进程实体的运行过程，是系统进行资源分配和调度的一个独立单位，符合统考标准定义。
- **II ✅** 创建进程的本质是创建 PCB，撤销进程的本质是回收 PCB。PCB 是进程存在的唯一标志。
- **III ✅** 进程的四大特征为动态性（最根本）、并发性、独立性和异步性。
- **IV ✅** 同一进程内线程切换不涉及地址空间切换，不会引起进程切换；跨进程的线程切换必然触发进程切换。

---

**02. B**

- **B ✅** 阻塞态只能由运行态转换而来（进程主动执行阻塞原语），从阻塞态出发只能转换到就绪态（等待事件发生后被动唤醒），不能直接转换为运行态。
- A ❌ 运行态可由就绪态转换而来，可转换到就绪态（时间片到/被抢占）、阻塞态（主动）或终止态。
- C ❌ 就绪态可由运行态或阻塞态转换而来，但就绪态可转换到运行态（被调度选中），不符合"只能转换到就绪态"。
- D ❌ 终止态只能从运行态进入，转换后进程结束，不能转换到任何其他状态。

---

**03. D**

- A ✅ 进程是资源分配的基本单位，线程是 CPU 调度的基本单位。
- B ✅ 同一进程内线程切换只涉及寄存器等少量上下文的保存与恢复，不涉及地址空间切换，开销远小于进程切换。
- C ✅ 引入线程后，进程"执行"状态实际是指其某个线程正在运行。
- **D ❌** 同一进程内的多个线程共享该进程的完整地址空间，可以直接通过共享变量通信，无需借助 IPC 机制。

---

**04. D**

PCB 包含四类信息：进程标识信息（如 PID、PPID、UID）、处理机状态信息（如通用寄存器、PC、PSW、SP）、调度信息（如进程状态、优先级、CPU 占用时间）、资源和内存信息（如代码段指针、数据段指针、文件描述符）。函数调用栈中的局部变量存储在线程的私有栈中，不属于 PCB。

---

**05. D**

- **I ✅** FCFS 对长作业（CPU 繁忙型）有利，短作业（I/O 繁忙型）排到队尾需重新等待，等待时间变长。
- **II ✅** SJF 的重要性质：所有作业同时到达时，SJF 的平均周转时间最小。
- **III ✅** 响应比公式正确，短作业等待时间短时响应比高；长作业等待时间增长后响应比提高，避免饥饿。
- **IV ✅** 时间片过大 → 退化为 FCFS；时间片过小 → 频繁上下文切换 → 系统开销增大。

---

**06. A**

SJF 执行顺序：按运行时间从小到大（J2=1, J4=1, J3=2, J5=5, J1=10）。

调度甘特图：J2(0~1) → J4(1~2) → J3(2~4) → J5(4~9) → J1(9~19)

各作业周转时间：
| 作业 | 完成时间 | 周转时间 |
|:---:|:---:|:---:|
| J1 | 19 | 19 |
| J2 | 1 | 1 |
| J3 | 4 | 4 |
| J4 | 2 | 2 |
| J5 | 9 | 9 |

平均周转时间 = (19 + 1 + 4 + 2 + 9) / 5 = 35 / 5 = **7**

---

**07. B**

RR（时间片 = 1）的执行顺序（数字表示剩余时间）：

0-1: J1(10→9) → 1-2: J2(1→0完成,T=2)
2-3: J3(2→1) → 3-4: J4(1→0完成,T=4)
4-5: J5(5→4) → 5-6: J1(9→8)
6-7: J3(1→0完成,T=7) → 7-8: J5(4→3)
8-9: J1(8→7) → 9-10: J5(3→2)
10-11: J1(7→6) → 11-12: J5(2→1)
12-13: J1(6→5) → 13-14: J5(1→0完成,T=14)
14-19: J1(5→4→3→2→1→0完成,T=19)

各作业周转时间：
| 作业 | 完成时间 | 周转时间 |
|:---:|:---:|:---:|
| J1 | 19 | 19 |
| J2 | 2 | 2 |
| J3 | 7 | 7 |
| J4 | 4 | 4 |
| J5 | 14 | 14 |

平均周转时间 = (19 + 2 + 7 + 4 + 14) / 5 = 46 / 5 = **9.2**

**08. D**

多级反馈队列调度过程：

| 时间段 | 执行队列 | 执行进程 | 说明 |
|:---:|:---:|:---:|:---|
| 0~10ms | Q1 | P1 | P1 运行 10ms，剩余 80ms → 降入 Q2 |
| 10~30ms | Q2 | P1 | P1 运行 20ms，剩余 60ms → 降入 Q3 |
| t=30ms | | | P2 到达，进入 Q1。P1 在 Q3，Q1 > Q3，P2 抢占 CPU |
| 30~40ms | Q1 | P2 | P2 运行 10ms，剩余 20ms → 降入 Q2 |
| 40~60ms | Q2 | P2 | P2 运行 20ms，完成 |
| 60~90ms | Q3 | P1 | P1 运行 30ms，剩余 30ms → Q3 队尾 |
| 90~120ms | Q3 | P1 | P1 运行 30ms，完成 |

**09. C**

- **I ✅** 时间片用完 → 当前进程转为就绪态，需调度新进程。
- **II ✅** 进程因 I/O 阻塞 → 交出 CPU，需调度就绪队列中的进程。
- **III ✅** 进程执行结束 → 释放 CPU，需调度新进程。
- IV ❌ 新进程进入就绪队列只是增加了就绪进程的数量，并不是直接引起调度的原因——若当前进程仍在运行且未被抢占，则不会立即触发调度。

**10. A**

互斥信号量 mutex 初值为 1，表示临界资源可用。记录型信号量下：

P 操作：mutex--；若 mutex < 0，则进程阻塞。
V 操作：mutex++；若 mutex ≤ 0，则唤醒一个等待进程。

当前 mutex = -1：
- mutex < 0 表示有进程在等待，等待数为 |mutex| = 1。
- 从初值 1 到 -1，经历了 2 次 P 操作（1→0→-1），1 次 V 操作（-1→0→-1 若在原序列中）。实际上，当 mutex = -1 时：已有一个进程通过 P 操作成功进入临界区（mutex 从 1→0），另一个进程执行 P 操作后 mutex 从 0→-1 后被阻塞。所以临界区内有 1 个进程，等待队列中有 1 个进程。

因此 **A** 正确。

---

**11. C**

若互换 P 操作顺序，变为：
- 生产者：P(mutex) → P(empty) → ... → V(full) → V(mutex)
- 消费者：P(mutex) → P(full) → ... → V(empty) → V(mutex)

当缓冲区 **满**（empty = 0, full = n）时：
1. 生产者执行 P(mutex) 成功（mutex = 0），然后执行 P(empty) 时发现 empty = 0 而阻塞
2. 消费者想取数据，但必须先执行 P(mutex)，发现 mutex = 0 而阻塞
3. 双方死锁：生产者持 mutex 等空位，消费者等 mutex

当缓冲区 **空**（full = 0, empty = n）时：
1. 消费者执行 P(mutex) 成功（mutex = 0），然后执行 P(full) 时发现 full = 0 而阻塞
2. 生产者想放数据，但必须先执行 P(mutex)，发现 mutex = 0 而阻塞
3. 双方死锁：消费者持 mutex 等产品，生产者等 mutex

因此，缓冲区满和空时均可能死锁，**C** 正确。

**12. B**

- **I ✅** 信号量有整数值表示资源数；条件变量不维护数值，仅维护等待队列。
- **II ✅** 信号量 P 操作在资源不足时阻塞进程；管程中 wait 操作在条件不满足时阻塞进程。
- III ❌ 管程的 signal 操作仅在没有等待进程时不产生效果，但若有等待进程则唤醒一个。然而信号量的 V 操作 **总是会改变信号量的值**（S.value++），无论是否有等待进程。这是两者关键区别之一。
- **IV ✅** 管程中 signal 若无等待进程，则不产生任何效果。

---

**13. D**

- **I ✅** 互斥条件：资源被排他性使用，一次仅允许一个进程访问。
- **II ✅** 不可剥夺条件：进程已获得的资源在使用完毕前不能被强制剥夺。
- **III ✅** 请求并保持条件：进程在持有至少一个资源的同时提出新的资源请求。
- **IV ✅** 循环等待条件：存在进程集合 {P0, P1, ..., Pn}，其中 Pi 等待的资源被 P(i+1) mod (n+1) 占有。

四者同时满足时死锁才可能发生。

**14. A**

首先计算 Need = Max - Allocation：

| 进程 | Need |
|:---:|:---:|
| P0 | (7,4,3) |
| P1 | (1,2,2) |
| P2 | (6,0,0) |
| P3 | (0,1,1) |
| P4 | (4,3,1) |

Available = (3,3,2)

验证选项 A (P1 → P3 → P4 → P2 → P0)：
- P1: Need(1,2,2) ≤ (3,3,2) ✓ → Work = (3,3,2)+(2,0,0) = (5,3,2)
- P3: Need(0,1,1) ≤ (5,3,2) ✓ → Work = (5,3,2)+(2,1,1) = (7,4,3)
- P4: Need(4,3,1) ≤ (7,4,3) ✓ → Work = (7,4,3)+(0,0,2) = (7,4,5)
- P2: Need(6,0,0) ≤ (7,4,5) ✓ → Work = (7,4,5)+(3,0,2) = (10,4,7)
- P0: Need(7,4,3) ≤ (10,4,7) ✓ → Work = (10,4,7)+(0,1,0) = (10,5,7)
所有进程 Finish = true → 安全 ✓

B ❌ P0 第一步 Need(7,4,3) ≤ (3,3,2)? 7 > 3，不满足。
C ❌ P2 第二步 Need(6,0,0) ≤ (5,3,2)? 6 > 5，不满足。
D ❌ P4 第一步 Need(4,3,1) ≤ (3,3,2)? 4 > 3，不满足。

**A** 正确。

**15. C**

- A ✅ 死锁预防通过破坏必要条件之一实现，实现简单但资源利用率低（如静态分配导致资源闲置）。
- B ✅ 死锁避免通过银行家算法在分配前判断安全性，资源利用率较预防更高。
- **C ❌** 安全状态一定不会发生死锁 ✓，但不安全状态 **不一定** 会发生死锁——不安全状态只是可能演变为死锁，不一定必然发生。这是统考反复考查的核心区分。
- D ✅ 死锁检测允许系统进入死锁状态，发现后通过资源剥夺或终止进程等方式解除。
