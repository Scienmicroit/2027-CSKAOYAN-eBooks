## 二、内存管理的基本原理和要求

### 1. 内存管理的主要功能
| 功能 | 说明 |
|------|------|
| 内存空间的分配与回收 | 跟踪内存使用状态，分配/回收空间 |
| 地址转换 | 逻辑地址 → 物理地址 |
| 内存空间的扩充 | 虚拟存储技术 |
| 内存共享 | 多个进程安全访问同一物理内存区域 |
| 存储保护 | 确保各进程只访问自身授权区域 |

### 2. 程序的链接与装入

**三个步骤：编译 → 链接 → 装入**

**三种链接方式：**
| 链接方式 | 发生时机 | 特点 |
|----------|----------|------|
| 静态链接 | 运行前 | 各目标模块链接成完整装入模块，之后不再拆分 |
| 装入时动态链接 | 装入时 | 边装入边链接，便于修改和共享 |
| 运行时动态链接 | 执行时 | 需要时才加载链接，未使用的模块不装入 |

**三种装入方式：**
| 装入方式 | 适用环境 | 特点 |
|----------|----------|------|
| 绝对装入 | 单道程序 | 编译时直接生成绝对地址 |
| 可重定位装入（静态重定位） | 多道程序 | 装入时一次性转换所有地址，运行期间不能移动 |
| 动态运行时装入（动态重定位） | 多道程序 | 地址转换推迟到执行时，支持程序移动和紧凑 |

### 3. 内存保护
- **方法1**：上、下限寄存器
- **方法2**：重定位寄存器（基址寄存器）+ 界地址寄存器（限长寄存器）
  - 重定位寄存器：用于"加"，逻辑地址 + 基址 → 物理地址
  - 界地址寄存器：用于"比"，判断是否越界
- 加载必须通过**特权指令**完成，仅操作系统内核有此权限

### 4. 内存共享
- 只有**只读区域**（可重入代码/纯代码）可以共享
- 每个进程需配备独立的**私有数据区**
- 分页系统：共享代码的每个页框都需要每个进程的页表项指向
- 分段系统：只需一个段表项指向共享段

## 三、连续分配管理方式

### 1. 单一连续分配
| 特点 | 说明 |
|------|------|
| 内存划分 | 系统区（低地址）+ 用户区 |
| 优点 | 结构简单、无外部碎片、无需内存保护 |
| 缺点 | 仅适用单用户单任务、有内部碎片、利用率极低 |

### 2. 固定分区分配
| 特点 | 说明 |
|------|------|
| 划分方式 | 分区大小相等 / 分区大小不等 |
| 管理结构 | 分区使用表 |
| 优点 | 无外部碎片 |
| 缺点 | 有内部碎片、作业大于所有分区则无法装入、缺乏灵活性 |

### 3. 动态分区分配（可变分区分配）

**基本原理**：按进程实际需求动态分配大小匹配的连续空间

**外部碎片**：散布在已分配分区之间的小空闲块，可通过**紧凑技术**缓解

**内存回收四种情形：**
1. 回收区与前空闲分区相邻 → 合并
2. 回收区与后空闲分区相邻 → 合并
3. 回收区与前后均相邻 → 三者合并
4. 回收区与任何空闲分区不相邻 → 新建表项

#### 基于顺序搜索的分配算法

| 算法 | 排序方式 | 分配策略 | 特点 |
|------|----------|----------|------|
| **首次适应（FF）** | 地址递增 | 第一个满足请求的空闲分区 | 优先利用低地址，保留高地址大空间；低地址易积累小碎片 |
| **邻近适应（NF）** | 地址递增（循环） | 从上一次查找结束位置开始 | 减少低地址重复扫描，大空闲分布不均 |
| **最佳适应（BF）** | 容量递增 | 最小的可用分区 | 保留大空闲区，但产生难以利用的小碎片，性能通常较差 |
| **最坏适应（WF）** | 容量递减 | 最大的可用分区 | 避免小碎片，但很快难以满足大作业需求 |

**综合来看，首次适应算法最好。**

#### 基于索引搜索的分配算法
| 算法 | 特点 |
|------|------|
| 快速适应（Quick Fit） | 按固定尺寸分类，查找快、无碎片、但回收复杂 |
| 伙伴系统（Buddy System） | 分区大小为 2^k，回收时合并伙伴 |
| 哈希算法 | 以分区大小为关键字建哈希表 |

**易错点**：伙伴系统中，每次回收时仅合并大小相等的空闲分区（2024 统考真题）。

## 四、基本分页存储管理

### 1. 基本概念
- **页面**：进程逻辑地址空间划分的等长单元
- **页框（页帧/物理块）**：物理内存划分的等长单元（与页面大小相同）
- **页号** → **页框号** 映射通过**页表**实现
- 分页管理**无外部碎片**，有**页内碎片（内部碎片）**，平均约半页

### 2. 逻辑地址结构
```
| 页号 P (高位) | 页内偏移量 W (低位) |
```
- 页面大小通常为 2 的整数次幂（如 4KB）
- 例：32 位地址、4KB 页面 → 页号 20 位，页内偏移 12 位

### 3. 基本地址变换过程
1. 计算页号 P = A/L，页内偏移 W = A%L
2. 判断页号是否越界（P >= 页表长度 M → 越界中断）
3. 页表项地址 = 页表始址 F + P × 页表项长度，取出物理块号 b
4. 物理地址 E = b × L + W

**注意**：无快表时，每次访存需**两次访问内存**（先访问页表，再访问数据）。

### 4. 具有快表（TLB）的地址变换
- **快表命中**：一次访存即可
- **快表未命中**：需两次访存，并将页表项装入快表
- 基于局部性原理，命中率通常达 90% 以上

### 5. 两级（多级）页表
**解决的问题**：单级页表过长、占用内存过多
**带来的问题**：无快表时，地址变换需多次访存（两级页表需 3 次）

**地址变换过程（二级页表）：**
1. 页目录号 → 访问外层页表，获取内层页表物理起始地址
2. 二级页号 → 访问内层页表，获取物理块号
3. 物理块号 + 页内偏移 → 物理地址

## 五、基本分段存储管理

### 1. 分段管理的逻辑地址结构
```
| 段号 S | 段内偏移量 W |
```
- 地址空间是**二维的**
- 段长不固定，由程序逻辑结构决定

### 2. 段表
- 每个段表项记录：**段基址**（起始地址）+ **段长**
- 每个进程一张段表

### 3. 地址变换过程
1. 分离出段号 S 和段内偏移量 W
2. 判断段号是否越界（S >= 段表长度 M → 越界中断）
3. 判断段内偏移是否越界（W >= 段长 C → 段内越界中断）
4. 物理地址 E = 基址 b + W

### 4. 分页 vs 分段 对比

| 对比项 | 分页 | 分段 |
|--------|------|------|
| 目的 | 提高内存利用率（系统需要） | 满足用户编程、共享、保护等需求 |
| 长度 | 固定，由系统决定 | 不固定，由程序逻辑决定 |
| 地址空间 | 一维（线性） | 二维（段号+段内偏移） |
| 碎片 | 有内部碎片，无外部碎片 | 有外部碎片，无内部碎片 |
| 共享 | 共享需建立 N 个页表项（N 为页数） | 共享只需一个段表项 |
| 用户可见性 | 对用户透明 | 对用户可见（编译器处理） |

---

## 六、段页式存储管理

### 1. 基本结构
- 地址空间先分段（逻辑结构），每段再分页（物理管理）
- **逻辑地址结构**：段号 S + 页号 P + 页内偏移量 W
- 地址空间为**二维**（用户视角按段组织，分页对用户透明）

### 2. 数据结构
- 每个进程**一张段表**，每个段对应**一张页表**
- 段表项记录：页表始址 + 页表长度

### 3. 地址变换过程（无快表需 3 次访存）
1. 段号 → 段表 → 找到该段的页表始址
2. 页号 → 页表 → 找到物理块号
3. 物理块号 + 页内偏移 → 物理地址

### 4. 三种存储管理方式访存次数对比
| 管理方式 | 无快表访存次数 |
|----------|---------------|
| 分页 | 2 |
| 分段 | 2 |
| 段页式 | 3 |

## 七、虚拟内存基本概念

### 1. 传统存储管理方式的特征（缺点）
- **一次性**：作业必须全部装入内存后才能运行
- **驻留性**：作业一旦装入，一直驻留到结束

### 2. 局部性原理
| 类型 | 含义 | 例 |
|------|------|----|
| 时间局部性 | 某条指令/数据被访问后，不久很可能再次被访问 | 循环操作 |
| 空间局部性 | 某存储单元被访问后，附近单元也很可能被访问 | 顺序执行、数组 |

### 3. 虚拟存储器的定义与特征
**定义**：基于局部性原理，将外存作为内存的透明扩展，仅将当前所需页面调入内存，其余暂留外存，通过请求调页和页面置换机制实现。

**三大特征：**
| 特征 | 说明 |
|------|------|
| 多次性 | 无须一次性全部装入，可分多次调入 |
| 对换性 | 运行过程中无须常驻内存，可换出/换入 |
| 虚拟性 | 从逻辑上扩充内存容量 |

### 4. 虚拟内存的实现方式
- 请求分页存储管理（最常用）
- 请求分段存储管理
- 请求段页式存储管理

### 5. 硬件支持
- 足够容量的内存和外存
- 页表/段表机制
- 中断机构（缺页/缺段中断）
- 地址变换机构

## 八、请求分页管理方式

### 1. 页表项新增字段
| 字段 | 说明 |
|------|------|
| 状态位 P（存在位） | 标记该页是否已调入内存 |
| 访问字段 A | 记录访问次数或时间间隔，供置换算法参考 |
| 修改位 M（脏位） | 标记该页是否被修改过 |
| 外存地址 | 指示该页在外存中的存放位置 |

### 2. 缺页中断的特点
- 在**指令执行期间**触发（非指令执行完毕）
- 一条指令可能引发**多次**缺页中断（如 copy A to B 最多 6 次）
- 缺页中断处理后，**重新执行**引发缺页的那条指令

### 3. 请求分页地址变换过程
1. 先检索快表（TLB）：命中 → 取出物理块号，置访问位为1
2. 若快表未命中 → 访问页表：
   - 页面在内存 → 取出物理块号，装入快表
   - 页面不在内存 → 触发缺页中断，调入页面，更新页表和快表
3. 物理块号 + 页内偏移 → 物理地址

## 九、页框分配

### 1. 最小页框数
- 能保证进程正常运行所需的最少页框数量
- 取决于**指令的格式、功能和寻址方式**
- 单地址直接寻址：至少 2 个页框（指令 + 数据）
- 间接寻址：至少 3 个页框
- 最坏情况：可能需 6 个页框

### 2. 驻留集大小
- 过小：缺页率升高
- 过大：浪费内存，降低并发能力

### 3. 内存分配与置换策略

| 策略 | 分配方式 | 置换方式 | 特点 |
|------|----------|----------|------|
| 固定分配局部置换 | 固定数量，不变 | 只从自身页框选 | 初始分配量难确定 |
| 可变分配全局置换 | 动态调整 | 可从所有页框选 | 灵活，但活跃进程可能膨胀 |
| 可变分配局部置换 | 动态调整 | 只从自身页框选 | 根据缺页率动态调整驻留集，效果最好 |

**注意**：固定分配 + 全局置换不可行（固定分配要求页框数量不变，全局置换会导致变化）。

### 4. 调入页面的时机
- **预调页策略**：预先调入相邻页面，预测成功率低（约50%），主要用于进程首次调入
- **请求调页策略**：缺页时调入，当前虚拟存储器大多采用

### 5. 从何处调入页面
- 有足够对换区：全部从对换区调入（速度快）
- 对换区不足：未修改页面从文件区调入，修改页面换出到对换区
- UNIX 方式：未运行页面从文件区调入，曾运行被换出页面从对换区调入

### 6. 如何调入页面
缺页中断 → 获取外存地址 → 判断内存是否已满：
- 未满：分配空闲页框 → 调入页面 → 更新页表
- 已满：选置换页 → 若修改位为1则写回 → 调入页面 → 更新页表

## 十、页面置换算法

### 1. 最佳置换算法（OPT）
- 淘汰**未来最长时间内不再被访问**的页面
- 理论上缺页率最低，但**无法实现**（无法预知未来）
- 用于评价其他算法的理论基准

### 2. 先进先出置换算法（FIFO）
- 淘汰**最早进入内存**的页面
- 实现简单，但性能较差（未利用局部性原理）
- 存在 **Belady 异常**：分配的物理块数增加，缺页次数反而可能上升
- 只有 FIFO 可能出现 Belady 异常

### 3. 最近最久未使用置换算法（LRU）
- 淘汰**最近最长时间未使用**的页面
- 性能接近 OPT，**不会出现 Belady 异常**
- 实现需硬件支持，开销较大
- 根据页面的"过去"判断（"向前看"），OPT 根据"未来"判断（"向后看"）

### 4. 时钟置换算法（CLOCK/NRU）

**（1）简单 CLOCK 算法：**
- 每个页面设置**访问位**（首次装入/被访问时置1）
- 所有页框组成循环队列，替换指针循环检查
- 规则：访问位=0 → 淘汰；访问位=1 → 置0，指针下移
- 也称**最近未用（NRU）算法**

**（2）改进型 CLOCK 算法：**
- 结合访问位 A 和修改位 M，页面分为四类：
| 类别 | A | M | 优先级（淘汰顺序） |
|------|---|---|-------------------|
| 1类 | 0 | 0 | 最佳淘汰页（优先） |
| 2类 | 0 | 1 | 次佳淘汰页 |
| 3类 | 1 | 0 | 可能再次被访问 |
| 4类 | 1 | 1 | 可能再次被访问（最末） |

**淘汰次序：** (0,0) → (0,1) → (1,0) → (1,1)

**扫描步骤：**
1. 第一轮：找 A=0,M=0（不修改A）
2. 若失败 → 第二轮：找 A=0,M=1（将经过页面的A置0）
3. 若失败 → 复位指针，所有A清零 → 重复第1步

### 5. 四种算法对比

| 算法 | 是否可实际实现 | Belady 异常 | 性能 | 实现开销 |
|------|--------------|-------------|------|----------|
| OPT | 否（理论） | 不会 | 最优 | — |
| FIFO | 是 | 会 | 较差 | 低 |
| LRU | 是（需硬件） | 不会 | 接近 OPT | 高 |
| CLOCK | 是 | 不会 | 较好 | 较低 |

### 6. 缺页率的快速计算方法
- 无论什么算法，**每种不同页面第一次访问时必然缺页**
- 缺页次数下限 = N（不同页号数）
- 缺页次数上限 = P（引用串总长度）

## 十一、抖动与工作集

### 1. 抖动（颠簸）
**定义**：刚刚换出的页面马上又要换入，刚刚换入的页面又立即被换出——频繁的页面调度行为。

**根本原因**：分配给进程的物理块数量过少，无法满足基本需求。

**后果**：磁盘 I/O 频率急剧上升，CPU 利用率趋近于零。

**解决措施**：
- 撤销部分进程（减少对内存的竞争）
- 调整驻留集大小

### 2. 工作集
**定义**：在某段时间间隔内，进程实际访问过的页面集合。

**工作集窗口**：工作集 W 由时间 t 和窗口尺寸 Δ 共同确定。

**重要结论**：驻留集大小不应小于工作集大小，否则进程将频繁缺页。

## 十二、页框回收（2025 新增考点）

### 页面缓冲算法
- **空闲页面链表**：未被修改的页面换出时，不写回磁盘，直接挂入空闲链表；后续访问相同页时可复用
- **修改页面链表**：脏页换出时先挂入链表，积累到一定数量后再批量写回磁盘
- **优点**：降低页面换入/换出频率，减少磁盘 I/O 开销
- 即使采用简单置换策略（如 FIFO）也能获得良好性能

## 十三、内存映射文件

**定义**：在磁盘文件与进程的虚拟地址空间之间建立直接映射关系。

**特点**：
- 像访问内存一样读写文件（无需传统文件 I/O 接口）
- 按需调页（首次访问时才加载页面）
- 进程退出或解除映射时，修改的页面才写回磁盘

**优点**：
1. 编程更简单
2. 便于多个进程共享同一个磁盘文件（通过页表映射到相同物理页）

**注意**：映射目标始终是进程的**虚拟地址空间**，不是物理地址空间。

## 十四、虚拟存储器性能影响因素

1. **页面大小**：大页面 → 页表短但碎片多；小页面 → 页表长但碎片少
2. **分配给进程的物理块数**：越多缺页率越低，但超过阈值后改善平缓
3. **页面置换算法**：好的算法（LRU、CLOCK）能有效降低缺页率
4. **写回磁盘频率**：批量写回优于每换一页即写回
5. **程序的局部化程度**：局部性越好，缺页率越低

## 十五、易错点与重要提示

1. **页表项地址 ≠ 页表项内容**：页表项地址 = 页表始址 + 页号 × 页表项长度；页表项内容是物理块号
2. **页表项大小**：需至少能表示所有物理页框号，通常设为 4B（含有效位等）
3. **缺页中断是内中断**，在指令执行期间触发，**不是**在指令执行完毕
4. **一条指令可能引发多次缺页中断**（如 copy 指令两个操作数各跨两页，最多 6 次）
5. **只有 FIFO 算法可能出现 Belady 异常**；OPT 和 LRU 不会
6. **固定分配 + 全局置换**不可行（二者互斥）
7. **工作集不一定是驻留集的子集**——工作集可能包含尚未调入或已被换出的页面
8. **段页式地址空间是二维的**（并非三维），因为分页对用户透明
9. **进程页表的起始地址存放在页表寄存器（PTR）**中，进程未执行时存放在 PCB 中
10. **虚拟存储器容量由虚拟地址位数决定**，而非内存或外存容量
11. **分页有内部碎片，无外部碎片；分段有外部碎片，无内部碎片**（段页式以分页为主，有内部碎片）
12. **首次适应算法不需要对空闲分区链排序**

## 十六、重要计算题型要点

### 1. 地址变换计算
- 页面大小 L → 页内偏移位数 = log2(L)
- 逻辑地址位数 = 页号位数 + 页内偏移位数
- 物理地址 = 块号 × L + 页内偏移

### 2. 多级页表级数计算
- 每页可容纳页表项数 = 页面大小 / 页表项大小
- 页号位数 / log2(每页可容纳页表项数) → 向上取整

### 3. 有效访问时间（EAT）计算
- EAT = 命中率 × 命中时间 + (1-命中率) × 未命中时间
- 考虑 TLB：EAT = h × (TLB_time + mem_time) + (1-h) × (TLB_time + n×mem_time)
  - h：TLB 命中率，n：访存次数（一级页表 n=2，二级 n=3...）

### 4. 缺页率与访存时间计算
- EAT = (1-p) × mem_time + p × (缺页处理时间 + mem_time)
- p：缺页率

### 5. 页面置换算法的缺页次数计算
- 初始状态：全空（或指定初始页面）
- 一步步模拟页面置换过程，记录缺页次数
