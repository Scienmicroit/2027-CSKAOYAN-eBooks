## 答案与解析

---

**01. C**

逻辑地址空间 256TB = 2^48B，页面大小 4KB = 2^12B，故页内偏移占 12 位，页号占 48 - 12 = 36 位。

页表项大小为 8B，一页（4KB）可容纳的页表项数 = 4KB / 8B = 2^12 / 2^3 = 2^9 = 512 个。每级页表用 9 位索引。

36 位页号 / 9 位每级 = 4 级。

- A ❌ **2 级只能覆盖 9×2=18 位页号，远小于 36 位**，无法完全覆盖地址空间，错误。
- B ❌ **3 级只能覆盖 9×3=27 位页号，仍小于 36 位**。若强行使用 3 级，每级页表将超过一页大小，违反题目条件，错误。
- **C ✅** 9×4=36 位页号被完全覆盖，每级大小恰好为一页，4 级是合理选择。
- D ❌ **5 级虽然也能覆盖 36 位页号，但多余层级不必要地增加了地址变换的访存次数**，不是最合理的级数。

> 陷阱解析：256TB 是 2^48B，不是 2^48 页。易误算为页号占 48 位。

**02. A**

单级页表有 TLB 情况下的地址变换流程：
1. 查询 TLB（耗时 t）
2. TLB 命中（概率 h）：直接获得物理块号，拼接页内偏移后访问一次内存取数据（耗时 T）
3. TLB 未命中（概率 1-h）：访问内存中的页表（耗时 T），获得物理块号，再访问一次内存取数据（耗时 T）

故 EAT = h × (t + T) + (1-h) × (t + 2T)

- **A ✅** 表达式完全正确。
- B ❌ **未计入 TLB 访问时间 t**。t 已在题目中明确给出，不可忽略，错误。
- C ❌ **TLB 未命中时用了 t + 3T**。单级页表 TLB 未命中只需 2 次访存（页表 1 次 + 数据 1 次），不是 3 次。3 次是二级页表无 TLB 时的访存次数，错误。
- D ❌ **同样错误地将 TLB 未命中时记为 3 次访存**，错误。

---

**03. C**

二级页表且未配置 TLB 时的地址变换过程：
1. 用页目录号访问外层页表（第 1 次访存），获得内层页表的物理起始地址
2. 用页号访问内层页表（第 2 次访存），获得物理块号
3. 物理块号与页内偏移拼接成物理地址，访问目标数据（第 3 次访存）

共需 **3 次**访存。

- A ❌ 1 次是 TLB 命中或直接物理寻址的情况，错误。
- B ❌ 2 次是**单级页表**无 TLB（页表 + 数据）或分段系统的访存次数，二级页表需要 3 次，错误。
- **C ✅** 3 次，正确。
- D ❌ 4 次是**三级页表**无 TLB 时的访存次数（3 级页表 + 数据）。二级页表只需 3 次，错误。

---

**04. C**

段页式系统无 TLB 时需要 3 次访存（段表 1 次 + 页表 1 次 + 数据 1 次）。TLB 命中时直接从 TLB 获得物理块号，只需访问 1 次内存获取数据。

- TLB 查找时间 t = 1ns
- 内存访问时间 T = 10ns
- TLB 命中率 h = 92%
- TLB 命中时总时间 = t + T = 1 + 10 = 11ns
- TLB 未命中时总时间 = t + 3T = 1 + 30 = 31ns

EAT = 0.92 × 11 + 0.08 × 31 = 10.12 + 2.48 = **12.6ns**

- A ❌ 10.0ns **相当于忽视 TLB 查找时间且假设全部命中**，错误。
- B ❌ 11.4ns **对应的 TLB 命中率为 98%**（0.98×11+0.02×31=11.4），但在命中率 92% 的条件下该值不正确。
- **C ✅** 12.6ns，正确。
- D ❌ 14.0ns **对应的 TLB 命中率约为 86.7%**，与实际 92% 不符，错误。

> 陷阱解析：段页式无 TLB 时应为 3 次访存，而非 2 次或 4 次。易与分页的 2 次或二级分页的 3 次混淆。

**05. C**

改进型 CLOCK 算法步骤：
- 第 1 轮：找 A=0,M=0（1 类），**不修改 A 位**。
- 第 2 轮：找 A=0,M=1（2 类），**将经过页面的 A 置 0**。
- 若前两轮失败（所有页面 A=1），复位指针，**所有 A 清零**，重复第 1 轮，此时必有可淘汰页。

- A ❌ **第 1 轮中不修改 A 位**，说"将经过页面的 A 置 0"错误。将 A 置 0 是第 2 轮的操作。
- B ❌ **第 2 轮中要将经过页面的 A 置为 0**，说"不修改任何页面的 A 位"错误。
- **C ✅** 前两轮失败说明全为 (1,x)，清零 A 后重复第 1 轮，此时所有页面的 A=0，必有 (0,0) 或 (0,1) 可淘汰，正确。
- D ❌ **改进型 CLOCK 优先淘汰 A=0,M=0 的页面（1 类），而非 (1,1)**。(1,1) 是优先级最低（最末淘汰）的页面，错误。

**06. B**

- A ❌ **固定分配的含义就是驻留集大小在运行期间固定不变**，不可动态调整。说"可动态调整"与定义矛盾。
- **B ✅** 可变分配全局置换中，缺页时若空闲页框已耗尽，允许从内存**所有**页框中选一页换出，不论归属哪个进程，**确实可能导致其他活跃进程的驻留集缩小**。正确。
- C ❌ **固定分配 + 全局置换不可行**——固定分配要求页框数不变，全局置换会导致页框数变化，二者互斥。说"可行"错误。
- D ❌ **缺页率过高时应增加页框（扩充驻留集）**，说"回收部分页框"恰好说反。回收是在**缺页率过低**时进行的操作。

**07. C**

- A ✅ 抖动的根本原因是**分配给进程的物理块数量过少**，正确。
- B ✅ 抖动时 CPU 忙于等待磁盘 I/O，利用率趋近于零，磁盘 I/O 频率急剧上升，正确。
- **C ❌** 采用更好的置换算法（如 LRU）可以**改善**缺页率，但**不能从根本上解决抖动**。只要物理块数量不足，无论用何种算法都会频繁缺页。**根本解决需要增加驻留集大小或减少竞争进程数**。本题选"错误的是"，故 C 为答案。
- D ✅ 撤销部分进程可减少内存竞争，是缓解抖动的有效措施，正确。

**08. A**

访问序列（共 21 次）：1, 4, 2, 3, 5, 3, 2, 2, 1, 1, 1, 3, 4, 5, 4, 4, 2, 1, 1, 3, 3

- **A ✅** 时刻 t₁ 指向第 5 次访问（页面 5），最近 Δ=5 次访问为：1,4,2,3,5（位置 1~5）。工作集 = {1,2,3,4,5}，大小 = 5。正确。
- B ❌ 时刻 t₂ 指向第 10 次访问（页面 1），最近 5 次访问为位置 6~10：3,2,2,1,1。工作集 = {1,2,3}，**大小为 3**，说"大小等于 5"错误。
- C ❌ **"一定越大"过于绝对**。若程序局部性极好，窗口增大但反复访问相同的少量页面，工作集大小可能不变。
- D ❌ 驻留集大小(3) < 工作集大小(5)时，进程应**频繁缺页**，说"一定不会发生缺页"恰好说反。

**09. B**

- A ❌ **空闲链表不清零**——页面缓冲算法的核心思想正是保留内容以供后续直接复用，清零会失去复用优势。说"必须清零"与算法思想相悖。
- **B ✅** 修改页面链表累积脏页后批量写回，减少 I/O 次数，降低换出开销。正确。
- C ❌ **未修改页面换出时直接挂入空闲链表，不写回磁盘**。说"先写回磁盘再挂入"凭空增加了不必要的 I/O 操作。
- D ❌ **即使采用简单置换策略（如 FIFO）也能获得良好性能**，无需与 LRU 绑定。说"必须与 LRU 配合"错误。

**10. D**

- A ❌ 建立映射时**不会立即加载文件内容**，是按需调页、首次访问时才调入。说"一次性全部加载"与虚拟内存按需调入思想相悖。
- B ❌ 内存映射文件将文件映射到进程的**虚拟地址空间**（而非物理地址空间），且**多个进程可以映射同一文件**（通过页表映射到相同物理页）。
- C ❌ 若页面被换出到外存对换区，**修改过的页面必须写回**（因为要腾出页框），不会拖延到解除映射。只有始终驻留内存的页面才在解除映射时写回。
- **D ✅** 按需调页，首次访问触发缺页中断，从磁盘调入目标页面。正确。

**11. B**

- A ❌ **并非任何访问序列下都会出现 Belady 异常**，只在特定序列（如 3,2,1,0,3,2,4,3,2,1,0,4）下才会发生。"在任何…都"表述过于绝对。
- **B ✅** OPT 算法淘汰未来最长时间不再访问的页面，是堆栈式算法（满足包含性质），**不会出现 Belady 异常**。正确。
- C ❌ **LRU 同样是堆栈式算法**，不会出现 Belady 异常。说"可能出现在特定序列下"错误。
- D ❌ **物理块从 m 增加到 m+1 时都可能出现 Belady 异常**，并非仅限于 3→4。如 2→3、4→5 都有可能。

**12. B**

逻辑地址空间大小为 2^16 **页**，即页号有 16 位。页面大小 2^10B，页表项大小 2B。

一页可容纳的页表项数 = 2^10 / 2 = 2^9 = 512 个。

每级页表用 9 位索引（2^9=512），页号共 16 位，16/9 = 1.78 → 向上取整，需要 **2 级页表**。

二级页表无 TLB：2 次访页表（外层 + 内层）+ 1 次访数据 = 3 次访存。

- A ❌ 2 次是**单级页表**无 TLB 的访存次数（页表 + 数据），不是二级页表。
- **B ✅** 3 次，正确。
- C ❌ 4 次是**三级页表**无 TLB 的访存次数（3 级页表 + 数据）。
- D ❌ 5 次是四级页表无 TLB 的访存次数。

> 陷阱解析：题目说"逻辑地址空间大小为 2^16 页"，即已告知页号 16 位，无需从页面大小推算页号位数。易误以页面大小 2^10B 直接作为页号位数计算。

**13. B**

分段地址变换的正确流程：
1. 分离段号 S 和段内偏移 W
2. 判断 S ≥ 段表长度 M？是 → 段号越界中断
3. 查段表得段长 C，判断 W ≥ C？是 → 段内越界中断
4. E = 段基址 b + W

- A ❌ 段号越界应触发**段号越界中断**，说"段内越界中断"混淆了两种不同的越界类型。
- **B ✅** 物理地址 = b + W，且应事先确认 W < C（段内未越界）。正确。
- C ❌ 段表项只记录**段基址和段长**，记录页表始址是**段页式**而非分段式的特征。
- D ❌ 顺序反了：**先判断段号越界，再判断段内偏移越界**。段号越界时该段根本不存在，无需检查段内偏移。

**14. D**

本题选"错误的是"。

- A ✅ 多级页表各级可以离散存放，不要求连续，正确。
- B ✅ 可将不常用的页表页放在外存，仅活跃页表页驻留内存，正确。
- C ✅ 多级页表需逐级查询，延长了地址变换时间，正确。
- **D ❌** 页表项**总数不变**——无论单级还是多级，页表项数量均由地址空间和页面大小决定（本题中均为 2^20 个），只是组织方式不同。说"总数少"错误。

**15. A**

32 位逻辑地址，页面 4KB=2^12B，页内偏移 12 位，页号 20 位。
页表项数 = 2^20，每个页表项 4B，页表大小 = 2^20 × 4B = 4MB。
每页 4KB，页表占 4MB / 4KB = **1024 个页面**。

改为二级页表：
- 每页可容纳页表项数 = 4KB / 4B = 1024 = 2^10 个
- 外层页表：1 页（含 2^10 个表项，指向最多 2^10 个内层页表页）
- 内层页表：最多 2^20 / 2^10 = 2^10 = 1024 页
- 总计最多 1 + 1024 = **1025 个页面**

- **A ✅** 单级 1024 页；二级最多 1025 页，正确。
- B ❌ **遗漏了外层页表的 1 页**，二级页表应比单级多 1 页（外层页表），说 1024 页错误。
- C ❌ **单级应为 1024 页**（2^20×4B/4KB=1024），说 1023 页计算错误。
- D ❌ **二级最多 1025 页**（1+1024），说 2048 页多算了将近一倍。

> 陷阱解析：二级页表比单级页表多**一页**（外层页表），而非相等或多一倍。易忽略外层页表占用。
