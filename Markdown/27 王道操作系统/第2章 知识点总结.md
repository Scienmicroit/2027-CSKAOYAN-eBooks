## 一、进程与线程

### 1.1 进程的定义与特征

**定义：** 进程是进程实体的运行过程，是系统进行**资源分配和调度**的一个独立单位。

**进程实体 = 程序段 + 数据段 + PCB**（创建进程的本质是创建 PCB）

**四大特征：**

| 特征 | 说明 |
|------|------|
| **动态性** | 有生命周期（最根本特征） |
| **并发性** | 多个进程可交替/并行执行 |
| **独立性** | 独立运行、获取资源的实体 |
| **异步性** | 推进速度不可预知 |

### 1.2 进程的组成与 PCB

| PCB 信息类别 | 内容 |
|------------|------|
| 进程标识 | PID、PPID、UID |
| 调度信息 | 状态、优先级、CPU 占用时间、阻塞原因 |
| 资源信息 | 代码/数据/堆栈段指针、文件描述符、I/O 清单 |
| CPU 上下文 | 通用寄存器、PC、PSW、SP |

**PCB 组织方式：** 链接方式（同状态 PCB 链成队列）、索引方式（状态索引表）

### 1.3 五状态模型

```
      创建态 → 就绪态 ⇄ 运行态 → 终止态
                  ↑        ↓
                  └─ 阻塞态 ←┘
```

| 转换 | 性质 |
|------|------|
| 就绪→运行 | 被调度选中 |
| 运行→就绪 | 时间片用完/被抢占 |
| 运行→阻塞 | **主动**（发起 I/O） |
| 阻塞→就绪 | **被动**（事件到达） |
| 阻塞→运行 | ❌ **不可能直接转换** |

### 1.4 进程通信（IPC）

| 方式 | 特点 |
|------|------|
| **共享存储** | 最快，需同步互斥 |
| **消息传递** | send/receive，支持直接/间接通信 |
| **管道** | FIFO 缓冲区，单向，亲缘进程间 |
| **信号** | 异步事件通知，内核态返回用户态时处理 |

### 1.5 线程

**线程 = 轻量级进程，程序执行流的最小单元，CPU 调度的基本单位**

| 对比 | 进程 | 线程 |
|------|------|------|
| 资源 | 资源分配单位 | 共享进程资源 |
| 调度 | 传统调度单位 | 独立调度基本单位 |
| 地址空间 | 独立 | 共享 |
| 切换开销 | 大 | 小 |

**两种实现方式：**

| | 用户级线程 ULT | 内核级线程 KLT |
|------|:--:|:--:|
| 管理 | 用户空间线程库 | 内核管理 |
| 切换开销 | 小 | 大（需切内核态） |
| 阻塞影响 | **一个阻塞→全进程阻塞** | 互不影响 |
| 多核并行 | 不能 | 能 |

**三种映射模型：** 多对一（一个阻塞全挂）、一对一（开销大）、多对多（折中）

## 二、CPU 调度

### 2.1 三级调度

| 层次 | 别名 | 频率 |
|------|------|------|
| 高级调度 | 作业调度 | 最低 |
| 中级调度 | 内存调度（挂起/换入换出） | 中等 |
| 低级调度 | 进程调度 | 最高（几十 ms） |

### 2.2 调度算法速查

| 算法 | 抢占 | 核心特点 |
|------|:--:|------|
| **FCFS** | 否 | 公平简单，短作业不利 |
| **SJF/SPF** | 可 | 平均等待时间最优，长作业饥饿 |
| **高响应比** | 否 | 响应比 =（等待+运行）/运行，兼顾长短 |
| **RR** | 可 | 时间片过大→FCFS，过小→切换频繁 |
| **多级反馈队列** | 可 | 新进程高优先级短时间片，降级加长时间片 |

### 2.3 调度指标

| 指标 | 公式 |
|------|------|
| 周转时间 | 完成时间 - 提交时间 |
| 带权周转时间 | 周转时间 / 实际运行时间 |
| 等待时间 | 就绪队列中等待 CPU 的总时间 -->

### 2.4 上下文切换

**切换 ≠ 调度：** 调度是决策（选谁），切换是执行（换过去）

**模式切换 ≠ 上下文切换：** 模式切换是同进程内用户态↔内核态；上下文切换是进程间切换

## 三、同步与互斥 ★★★

### 3.1 核心概念

| 概念 | 定义 |
|------|------|
| **临界资源** | 同一时刻仅允许一个进程访问 |
| **临界区** | 访问临界资源的代码段 |
| **同步** | 直接制约（协调运行次序） |
| **互斥** | 间接制约（竞争临界资源） |

**同步机制四准则：** 空闲让进、忙则等待、有限等待、让权等待（非必须）

### 3.2 实现方法

| 方法 | 代表 | 问题 |
|------|------|------|
| 软件-单标志法 | turn | 违背空闲让进 |
| 软件-双标志先检查 | flag[2] | 违背忙则等待 |
| 软件-双标志后检查 | flag[2] | 可能互相谦让 |
| **软件-Peterson** | flag[]+turn | ✅ 满足前三，❌ 忙等待 |
| 硬件-中断屏蔽 | 关中断 | 单 CPU 可用，多 CPU 失效 |
| 硬件-TS/Swap | 原子指令 | 忙等待 |

### 3.3 信号量 ★★★

```
P(S): S.value--; if S.value < 0 → block
V(S): S.value++; if S.value ≤ 0 → wakeup
```

| 用途 | 初值 | 模式 |
|------|------|------|
| 互斥 | 1 | P(mutex) ... V(mutex) 夹临界区 |
| 同步 | 0 | 先执行者 V，后执行者 P |
| 前驱 | 0 | 每条前驱边一个信号量 |

> **口诀：** 要用什么 P 一下，提供什么 V 一下，互斥夹紧

### 3.4 经典同步问题

| 问题 | 关键信号量 | 核心技巧 |
|------|-----------|---------|
| 生产者-消费者 | empty=n, full=0, mutex=1 | **先 P 资源再 P 互斥**，否则死锁 |
| 读者-写者 | rw=1, mutex=1, count | count 计读者数，首读 P(rw)，末读 V(rw) |
| 哲学家进餐 | chopstick[5]={1} | 防死锁：限 4 人/奇偶拿筷顺序/同时拿两筷 |

### 3.5 管程

- 封装共享数据 + 操作过程，任一时刻仅一个进程进入
- 条件变量：`x.wait()` 阻塞释放管程，`x.signal()` 唤醒
- 与信号量区别：条件变量不维护数值，只排队通知

## 四、死锁 ★★★

### 4.1 四个必要条件

| 条件 | 破坏方法 |
|------|---------|
| 互斥 | 改为共享（临界资源不可行） |
| 不可剥夺 | 请求失败强制释放（复杂） |
| 请求并保持 | 一次性申请全部（资源浪费） |
| 循环等待 | 资源有序分配（编号递增） |

### 4.2 银行家算法（死锁避免）

**核心数据结构：** Available、Max、Allocation、**Need = Max - Allocation**

**步骤：** 请求合法性检查 → 可用性检查 → 试探分配 → 安全性算法 → 正式分配/撤销

**安全性算法：** 找 Need ≤ Work 的进程，Work += Allocation，循环直至全部 Finish

> 🔑 **安全状态 → 一定无死锁；不安全状态 ⇏ 一定有死锁**

### 4.3 死锁检测与解除

- **死锁定理：** 资源分配图不可完全简化 ⟺ 死锁
- **解除：** 资源剥夺、撤销进程、进程回退

> 🔑 资源分配图有环 ⇏ 死锁（某类资源多实例时，环不必然死锁）

## 五、高频易错点

1. **就绪 vs 阻塞**：就绪 = 仅缺 CPU；阻塞 = 缺其他资源/等事件
2. **阻塞→运行**：不存在，必须先就绪
3. **运行→阻塞是主动**，阻塞→就绪是被动
4. **ULT 致命缺陷**：一个线程阻塞 → 全进程阻塞（内核只看到进程）
5. **PV 顺序**：先 P 资源信号量再 P 互斥信号量，颠倒可能死锁
6. **信号量值**：> 0 可用资源数；< 0 等待进程数（绝对值）
7. **不安全 ≠ 死锁**，资源分配图有环 ≠ 死锁
8. **死锁 vs 饥饿**：死锁必阻塞且涉及 ≥ 2 进程；饥饿可单进程可就绪
9. **Peterson**：满足空闲让进/忙则等待/有限等待，不满足让权等待（忙等）
10. **银行家算法**：Need = Max - Allocation，安全性算法用 Work 向量迭代
