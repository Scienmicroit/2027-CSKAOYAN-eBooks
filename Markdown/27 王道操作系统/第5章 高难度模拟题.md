## 答案与解析

---

**01. A（仅 I、II、IV）**

- **I ✅** DMA 的地址递增和计数递减由 DMA 控制器硬件自动完成；通道通过执行通道程序中的指令序列动态完成——这是两者最本质的区别之一。
- **II ✅** DMA 传输前，CPU 需初始化 MAR 和 DC 等参数；通道只需 CPU 发一条 I/O 指令指明通道程序入口，通道执行程序自主确定参数——通道程序的灵活性远高于 DMA。
- III ❌ 表述完全颠倒。DMA 处理的是**同一台设备的数据块传输**（数据在设备和内存之间）；通道可以处理**多台设备之间的数据传输**以及涉及多设备、多阶段的复杂 I/O 任务。
- **IV ✅** 一个 DMA 控制器通常对应于一台高速设备；一个通道可以通过多个控制器管理多台设备，这是通道优于 DMA 的重要方面。

---

**02. C**

- A ❌ 两者并非完全相同。核心区别在于折返时机不同。
- B ❌ 完全颠倒。**LOOK** 在到达当前方向**最远请求处**折返；**SCAN（传统电梯算法）** 在到达磁盘**物理末端**折返。B 选项将二者的行为说反了。
- **C ✅** 精确区分：LOOK 在方向上最远请求处折返（不走到物理边界）；SCAN 严格走到磁盘物理末端再折返（统考教材中 SCAN 定义为"单向到底再折返"）。注意，在实际系统中的 SCAN 实现常优化为 LOOK，但严格概念上二者不同。
- D ❌ 折返时机不同是核心区别，与服务策略无关。两者在折返后都沿反方向继续服务。

---

**03. C**

- A ❌ "in 追上 out"表示输入进程写入速度**快于**计算进程的处理速度，所有缓冲区被填满（全满），而不是全空。输入进程因无空缓冲区而阻塞，此时系统"受计算限制"。
- B ❌ "out 追上 in"表示计算进程处理速度**快于**输入进程的供给速度，所有缓冲区变空（全空），计算进程因无数据可处理而阻塞——此时系统"受 I/O 限制"。
- **C ✅** 精确表述：
  - **in 追上 out**：输入快于处理 → 全满缓冲区 → 输入进程阻塞（系统受计算限制）
  - **out 追上 in**：处理快于输入 → 全空缓冲区 → 计算进程阻塞（系统受 I/O 限制）
- D ❌ 指针追及时不是死锁，而是**正常同步现象**，通过阻塞较快的进程来协调速度差异，无需中断进程。死锁涉及资源的循环等待且无外力无法解除，而缓冲区追及可通过被阻塞方等待对方推进来自动恢复。

---

**04. C**

- A ✅ "假脱机"的"假"正是相对于早期"真"脱机（使用物理外围计算机）而言的——现代 SPOOLing 用软件进程模拟外围机，故称"假脱机"。
- B ✅ SPOOLing 的核心优点：用户进程写入输出井即认为 I/O 完成，可继续执行后续代码，实际物理打印由后台进程处理，对用户完全透明。
- **C ❌** 高精度陷阱。SPOOLing 技术中，物理设备**并未分配给任何进程**——教材明确写道："实际上并未将物理设备分配给任何进程"。进程只是使用输出井中的逻辑空间独占各自区域，物理打印机由后台输出进程独占使用。因此"物理设备已分配给了多个进程"是错误的。
- D ✅ 以磁盘空间换 CPU 时间，确属空间换时间技术。

---

**05. B**

设备分配的完整顺序为：**SDT → DCT → COCT → CHCT**。

流程解析：
1. **SDT（系统设备表）**：全局一张，根据设备类型或物理名查得目标设备的 DCT 指针
2. **DCT（设备控制表）**：每设备一张，判断设备状态（忙/闲），若空闲则分配；DCT 中包含指向控制器表的指针
3. **COCT（控制器控制表）**：每控制器一张，检查控制器状态，若空闲则分配；COCT 中包含指向通道表的指针
4. **CHCT（通道控制表）**：每通道一张，检查通道状态，若空闲则分配

> 四步全部成功才算分配完成。顺序不可调换：先找到设备，再找其所属控制器，再找控制器所属的通道。

---

**06. A（仅 I、II、IV）**

- **I ✅** 单缓冲公式 Max(T, C) + M 的核心假设正是 CPU 处理前一块数据（C）与设备向缓冲区写入下一块数据（T）可重叠执行。
- **II ✅** 双缓冲公式 Max(T, C+M) 的核心假设正是设备向一个缓冲写入（T）与操作系统传送 + CPU 处理另一个缓冲（C+M）可重叠执行。
- III ❌ 陷阱。当 T 远大于 C+M 时，双缓冲处理时间趋近于 **T**（因为 Max(T, C+M) = T），而**不是 T+M**。当 T > C+M 时，设备是瓶颈，设备可连续填充缓冲区，处理每块的时间由 T 决定，不需要额外加 M——因为 M 已经包含在 C+M 中比较了，既然 T 胜出，说明 C+M < T，M 已被 T 完全掩盖。
- **IV ✅** 当 C = 0 时：单缓冲 = Max(0, T) + M = T + M；双缓冲 = Max(T, 0+M) = Max(T, M)。完全正确。

---

**07. C**

- A ❌ C-SCAN 在**回扫过程中不服务任何请求**——这正是 C-SCAN 区别于 SCAN 的最核心特征。如果在回扫过程中也服务，那就退化成了 SCAN（双向服务）。
- B ❌ C-SCAN 到达物理末端后**不是立即折返**（立即折返是 SCAN 或 LOOK 的行为），而是**回扫到磁盘起始端**，再重新沿正向服务。C-SCAN 的核心特征是"单向服务 + 快速回扫（不服务）"。
- **C ✅** 精确表述：C-SCAN 回扫过程中不服务任何请求，回扫到起始端后重新沿正向服务所有请求。
- D ❌ C-SCAN 与 SCAN 的区别核心在于**服务方向策略（单向 vs 双向）**和**回扫行为**，而非回扫速度。实际机械运动中回扫速度由物理因素决定，与算法无关。

---

**08. A**

- **A ✅** 磁盘地址编码的正确顺序：**柱面号（柱面）→ 盘面号（磁头）→ 扇区号**。数据存放策略：先在同一个柱面的不同盘面（不同磁头）上存取所有扇区，装满后再切换至下一柱面。这种顺序可减少磁头臂的移动次数（因为切换柱面需要机械寻道，而切换盘面仅需电子切换磁头）。
- B ❌ 地址编码的**标准表示顺序**是（柱面号, 盘面号, 扇区号），而非（扇区号, 盘面号, 柱面号）。尽管扇区是磁盘上最小的可寻址单元，但编址时柱面号在最前面。
- C ❌ 磁盘地址必须包含柱面、盘面、扇区三维信息才能唯一定位一个扇区。"每个扇区独立编号"的说法不符合磁盘的物理寻址方式。
- D ❌ 填满顺序错误。数据存储策略是：先**填满同一柱面的所有盘面的扇区**，再转到下一柱面，而非"填满所有柱面再换盘面"。此外地址编码顺序也错，正确的是（柱面号, 盘面号, 扇区号），扇区号在最后而非中间。

---

**09. D（全部正确）**

- **I ✅** SSD 中，读写以**页**为单位，但擦除以**块**（包含多页）为单位。随机写时，如果目标页所在块中存在有效数据，需要将整个块读入缓存，更改目标页，然后擦除整个块，最后将整个块写回——这就是"读-改-擦-写"过程，比随机读慢 1~2 个数量级。
- **II ✅** SSD 无机械结构，无寻道时间和旋转延迟，因此不需要复杂的磁盘调度算法，简单的 FCFS 即可。
- **III ✅** SSD 无机械寻道和旋转延迟，随机读和顺序读速度都很高且差距很小。但随机写因涉及擦除操作而显著慢于顺序写（顺序写可合并为批量写入）。
- **IV ✅** SSD 的闪存单元有有限擦写次数（P/E 周期），需要磨损均衡使各块擦除次数均匀分布。机械磁盘没有擦写次数限制瓶颈，因此不需要磨损均衡。

**10. B**

- A ❌ 陷阱。共享设备**不是**"同一时刻"同时访问。"同时"应理解为"一段时间内"——多个进程的 I/O 请求在微观上通过调度分时交替执行。磁盘是共享设备，但任一时刻只有一个进程的 I/O 请求在执行。
- **B ✅** 精确定义：共享设备允许多个进程在一段时间内分时交替访问，但**任一时刻最多只有一个进程**在真正进行 I/O 操作。
- C ❌ 根本区别在于**是否能被多个进程在一段时间内分时交替使用**而非"同时访问"，且"同时访问"本身表述有误导。
- D ❌ SPOOLing 改造后，多个进程"看似"同时使用打印机，但物理打印仍然是**串行**的——后台输出进程按队列依次打印，并非物理上同时发送数据。

**11. B**

- A ❌ 完全颠倒。**缓冲区**中的数据是正在传输过程中的数据，用完即释放；**高速缓存**中的数据是已有数据的副本，可长期驻留（直到被替换算法替换）。
- **B ✅** 精确对比：高速缓存保存慢速设备上已有数据的副本，目的为**提高重复访问效率**；缓冲区暂存传输途中的数据，目的是**协调速度差异、减少中断频率、匹配数据粒度**。两者的数据生命周期和核心目的都不同。
- C ❌ 两者核心目的不同。高速缓存的目的是**提高重复访问效率**（避免频繁访问慢速设备）；缓冲区的目的是**提高数据传输效率与并行性**。尽管两者都位于高速设备和低速设备之间，但目的不同。
- D ❌ 磁盘高速缓存**不是硬件高速缓存**，教材明确写道："磁盘高速缓存不同于通常意义下的位于 CPU 与主存之间的硬件高速缓存，而是指利用内存中的存储空间暂存从磁盘读取的盘块数据。"

**12. A**

T = 100 μs, M = 10 μs, C = 30 μs。

**单缓冲处理时间** = Max(T, C) + M = Max(100, 30) + 10 = 100 + 10 = **110 μs**。

**双缓冲处理时间** = Max(T, C+M) = Max(100, 30+10) = Max(100, 40) = **100 μs**。

因此从 110 μs 变为 100 μs，选 **A**。

> 关键解析：当 T > C+M 时，单缓冲中 M 无法与 T 重叠（因为只有一个缓冲区，CPU 处理完前一块后必须等 M 完成才能释放缓冲区给设备），而双缓冲中 M 可以与 T 重叠（两个缓冲区轮换），因此双缓冲节省了 M 的时间开销。

**13. A（仅 I、II、III）**

- **I ✅** 中断驱动方式每传输一字（节）即产生一次中断；DMA 方式中 CPU 仅在预处理和后处理阶段介入，整块传输期间不需 CPU 干预。
- **II ✅** 两者数据路径的核心区别：中断驱动方式数据经过 CPU 寄存器中转（设备→CPU寄存器→内存）；DMA 方式数据直接在设备和内存之间传输。
- **III ✅** 字符设备（键盘、打印机等）传输速率低，适合中断驱动方式；块设备（磁盘等）传输速率高、数据量大，适合 DMA 或通道方式。
- IV ❌ 前半句正确（DMA 传输期间硬件自动控制），但后半句"中断响应过程完全由硬件完成，不需要软件参与"错误。硬件仅负责**保存断点（PC/PSW）和切换模式**，而**保护/恢复通用寄存器、识别中断源、执行中断服务程序**都属于软件行为。中断响应过程是硬件和软件协作完成的。

**14. A**

排序请求序列：14, 37, 53（当前位置）, 65, 67, 98, 122, 124, 183。

**LOOK 算法（到当前方向最远请求 183 折返）：**
- 正向（53→183）：53→65(12), 65→67(2), 67→98(31), 98→122(24), 122→124(2), 124→183(59)，小计 **130**
- 折返（183→反向最远 14）：183→37(146), 37→14(23)，小计 **169**
- **LOOK 总计 = 130 + 169 = 299**

**SCAN 算法（到磁盘物理末端 199 折返）：**
- 正向（53→199）：53→65(12), 65→67(2), 67→98(31), 98→122(24), 122→124(2), 124→183(59), 183→199(16)，小计 **146**
- 折返（199→反向最远 14）：199→14(185)
- **SCAN 总计 = 146 + 185 = 331**

因此 LOOK(299) < SCAN(331)，LOOK 避免了走到物理末端 199 的额外开销（多走了 199−183=16，折返又多走了 199−183=16，共节省 32）。

**15. B（仅 II、III、IV）**

- I ❌ 陷阱。设备驱动程序**向设备控制器发送命令和读取设备状态都必须处于内核态**，因为对 I/O 端口（寄存器）的访问属于特权操作，需要利用操作系统的 I/O 指令（如 IN/OUT 指令）完成，无法在用户态直接完成。即便采用内存映射 I/O（统一编址）方式，访问映射到 I/O 地址空间的内存区域也需要在内核状态下进行，因为普通用户程序无权绕过操作系统的虚拟内存保护直接访问物理内存。
- **II ✅** 教材明确说明："由于运行在中断上下文中，处理程序必须快速、非阻塞，不得调用可能引起进程睡眠或调度的函数。"
- **III ✅** 设备独立性软件负责逻辑设备名到物理设备名的映射（通过 LUT）、设备分配与回收、缓冲管理、差错控制、提供统一块大小等公共功能。
- **IV ✅** 用户层软件（库函数）运行在用户态，必须通过系统调用（Trap 指令陷入内核）才能获取操作系统服务。库函数本身不在内核态运行。
