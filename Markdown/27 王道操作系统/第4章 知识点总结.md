## 一、文件系统基础

### 1.1 文件三要素

1. **数据** — 实际存储的内容
2. **分类和索引信息** — 用于组织管理
3. **访问控制信息** — 权限保护

### 1.2 文件逻辑结构

| 类型 | 特点 |
|------|------|
| **无结构文件（流式）** | 字符流，只能顺序查找 |
| **有结构文件（记录式）** | 定长/变长记录 |

四种组织形式：

| 组织形式 | 特点 |
|---------|------|
| **顺序文件** | 串结构（按时间）/ 顺序结构（按键字排序）；定长时可折半查找 |
| **索引文件** | 为变长记录建索引表，支持随机访问；存储开销大 |
| **索引顺序文件** | 分组建索引，查找次数最优值为分块查找，m=√N |
| **直接文件** | 散列函数计算地址，需处理冲突 |

### 1.3 文件物理结构（分配方式）★★★

| 特性 | 连续分配 | 隐式链接 | 显式链接(FAT) | 索引分配 |
|------|:--:|:--:|:--:|:--:|
| 随机访问 | ✅ | ❌ | ✅ | ✅ |
| 外部碎片 | 有 | 无 | 无 | 无 |
| 文件扩展 | 难 | 易 | 易 | 易 |
| 存储开销 | 最小 | 指针 | FAT 常驻内存 | 索引块 |

#### 混合索引分配（UNIX inode，必考计算）

| 地址项 | 数量 | 数据量（4KB块/4B地址） |
|-------|------|----------------------|
| 直接地址 | 10 | 40KB |
| 一次间接 | 1 | 1024块 → 4MB |
| 二次间接 | 1 | 1024²块 → 4GB |
| 三次间接 | 1 | 1024³块 → 4TB |

> 🔑 先判断偏移量落在哪级，再计算访盘次数：直接 1 次，一级 2 次，二级 3 次，三级 4 次

## 二、目录与文件

### 2.1 FCB 与 inode

| | FCB | 索引节点(inode) |
|------|------|------|
| 内容 | 全部文件信息 | 文件名分离出去，目录项仅含文件名+inode号 |
| 检索效率 | 低（FCB大） | 高（目录项小，每块可存更多项） |

### 2.2 目录结构

| 类型 | 特点 |
|------|------|
| 单级目录 | 不允许重名 |
| 两级目录 | MFD + UFD，允许重名但不支持分类 |
| **树形目录** | 绝对/相对路径，当前工作目录 |
| 无环图目录 | 支持共享，共享计数器 |

### 2.3 文件打开过程

> 🔑 open() 之后使用**文件描述符(fd)**，不再使用文件名

| 表 | 作用域 |
|------|------|
| 系统打开文件表 | 全局，含打开计数器 |
| 进程打开文件表 | 每进程私有，含读写指针/权限 |

### 2.4 硬链接 vs 软链接 ★★

| | 硬链接 | 软链接 |
|------|------|------|
| 方式 | 多个目录项指向同一 inode | LINK 文件存路径名 |
| 原文件删除 | 仍可访问（count>0） | 访问失败 |
| 占用 inode | 否 | 是 |
| 查找速度 | 快 | 慢（路径解析） |
| 链接计数 | 影响 | 不影响 |

## 三、文件系统

### 3.1 磁盘布局

```
MBR(0号扇区) → 引导块 → 超级块 → 空闲空间 → i节点区 → 根目录 → 数据区
```

### 3.2 外存空闲空间管理

| 方法 | 核心 | 考点 |
|------|------|------|
| 空闲表法 | 类似动态分区 | 分配效率高 |
| 空闲盘块链 | 单块链接 | 链表长，效率低 |
| 空闲盘区链 | 连续盘区链接 | 较高效 |
| **位示图法** | 1位/盘块 | `b = n(i-1) + j` |
| 成组链接法 | UNIX 分组索引 | 栈顶分配，栈满写入新索引块 |

> 🔑 位示图占用空间只与外存总容量有关，与空闲块数量无关

### 3.3 虚拟文件系统 VFS

- 仅存在于**内存**，不占磁盘——不是真正的文件系统
- 向上提供统一接口，屏蔽 ext4/NTFS/FAT 差异
- 四种对象：超级块对象、V节点对象(vnode)、目录项对象(dentry)、文件对象

## 四、高频易错点

1. **逻辑结构 vs 物理结构**：前者是用户视角"是什么"，后者是"怎么存"
2. **隐式链接不支持随机访问**，显式链接(FAT)支持（FAT在内存中查表）
3. **open() 后只用 fd，不用文件名**
4. **硬链接 count=0 才释放**；软链接删原文件即失效
5. **混合索引计算**：先确定偏移量在哪级，再算访盘次数
6. **VFS 不存数据**，只是内存中的抽象层
7. **连续分配文件长度固定**，插入需大量移动 → I/O 次数最多
8. 磁盘 inode 存于磁盘，内存 inode 是文件打开时载入
