## 答案与解析

---

**01. B**

- A ❌ 对于定长按键字排序的顺序文件，支持通过偏移量直接定位记录（$A_i = iL$），因此支持随机访问。顺序文件不支持随机访问的情况仅针对变长记录或存储于磁带等顺序存储设备。
- **B ✅** 索引文件为每条记录建立一个索引项，支持随机存取，但索引表本身需要额外存储空间，空间开销大。
- C ❌ 索引顺序文件中，记录按组划分，组间按键字有序，但**组内记录可以无序**，这是索引顺序文件区别于顺序文件的重要特征。
- D ❌ 直接文件（散列文件）通过散列函数计算地址，关键字可能映射到同一地址，从而引发冲突，需要处理冲突。

---

**02. B**

N = 10000，最优分组数 $m = \sqrt{N} = 100$，每组记录数 $s = N/m = 100$。

平均查找次数 = 索引表查找次数 + 组内查找次数 = $m/2 + s/2 = 100/2 + 100/2 = 100$ 次。

- A ❌ 50 次对应误用 $m/2$ 或 $s/2$ 单方面计算。
- **B ✅** 正确结果。
- C ❌ 150 次可能误用 $\sqrt{N} + \sqrt{N} = 200/2$... 非标准。
- D ❌ 500 次对应未分组时顺序查找的平均次数。

---

**03. C**

- A ✅ 连续分配支持随机访问（计算起始块号+偏移量），但产生外部碎片，文件扩展时可能需要移动大量数据。
- B ✅ 隐式链接的指针存放在每个盘块中，因此访问第 i 块必须从首块开始顺序读取至第 i 块，不支持随机访问。
- **C ❌** 显式链接分配（FAT）的**核心优势是 FAT 在系统启动时即被加载到内存并常驻**，后续查找在内存中完成，无需每次都从磁盘读取。选项说"存储在磁盘上，每次访问需读取 FAT 到内存"完全错误。
- D ✅ 索引分配无外部碎片，扩展方便，但需要索引块存储指针信息。

---

**04. B**

- A ❌ FCB 包含文件名、物理位置、权限信息等全部文件信息；inode 将文件名分离出去，仅存储除文件名外的描述信息。二者内容不同。
- **B ✅** inode 将文件名分离，目录项仅包含文件名和 inode 编号，尺寸更小。同样大小的盘块可存放更多目录项，检索时所需磁盘 I/O 次数减少。
- C ❌ inode 分为**磁盘 inode**（在磁盘上持久存储）和**内存 inode**（文件打开时载入内存）。磁盘上一定存在 inode。
- D ❌ 文件名存储在**目录项**中，不在 inode 中。inode 存储的是除文件名外的文件属性信息。

---

**05. C**

- A ❌ 绝对路径是从**根目录**开始的完整路径名，不是当前目录。
- B ❌ 相对路径是从**当前工作目录**开始的路径名。
- **C ✅** 树形目录结构中，每个文件有唯一的绝对路径，从根目录到该文件。
- D ❌ 树形目录结构**允许**不同目录下的文件使用相同文件名。

---

**06. C**

- A ✅ 硬链接创建新的目录项指向相同的 inode。
- B ✅ 软链接是一个新文件（LINK 类型），系统为其分配一个新的 inode，inode 中存储目标文件的路径名。
- **C ❌** 硬链接与"原文件"地位对等，删除原文件只是将 inode 的链接计数减 1。只要 count > 0，文件数据仍然存在，可通过其他硬链接正常访问。只有所有链接都被删除（count = 0）时，数据才被回收。
- D ✅ 硬链接直接通过 inode 指针访问，软链接需要逐级解析路径名，且软链接本身是独立文件（一次间接寻址），速度更慢。

---

**07. A**

- F1 初始链接计数 = 1。
- 为 F1 创建硬链接 F2：在 F1 的目录中增加一个指向 F1 inode 的目录项，inode 链接计数 +1 → 2。**硬链接影响 inode 链接计数。**
- 为 F1 创建软链接 F3：创建一个新文件（新 inode），其中存放 F1 的路径名。**软链接不影响原文件的链接计数。**
- 因此 F1 链接计数 = 2，F3 链接计数 = 1（软链接新文件自身的链接计数为 1）。
- A ✅ 2；1。
- B ❌ 将软链接也影响链接计数。
- C ❌ 以为硬链接不影响计数。
- D ❌ 以为软链接有 0 个链接。

---

**08. D（全部正确）**

- **I ✅** open() 后将 fd 分配给进程，后续系统调用（read/write/lseek/close）均使用 fd，不再使用文件名。这是操作系统课程的常考重点。
- **II ✅** 系统打开文件表是全局唯一的数据结构，每个表项均包含一个打开计数器（open count），记录当前有多少进程打开了该文件。
- **III ✅** 当同一进程两次 open 同一文件时，获得两个不同的 fd，在系统打开文件表中创建两个独立的表项（各自有独立的文件读写指针），指向同一个 inode。
- **IV ✅** 文件读写指针（当前文件位置）是进程私有的，存储在进程打开文件表中，不同进程打开同一文件可拥有不同的读写位置。

---

**09. A**

- **A ✅** 空闲表法为每个连续空闲区建立一个表项，类似于动态分区分配，适用于连续分配方式。系统在创建文件时查找合适的空闲区分配。
- B ❌ 位示图法所需的存储空间只与**磁盘总容量**（总盘块数）有关，与空闲块数量**无关**。这是位示图法的关键特性。
- C ❌ 成组链接法中，当空闲栈非空时，直接从栈顶分配，无需访问磁盘。仅在栈空时需要读入新的分组块。
- D ❌ 空闲盘块链以单个盘块为单位链接，链表很长，查找效率低。

---

**10. D**

- A ✅ VFS 是内存中的抽象层，不是实际的文件系统，不占用磁盘空间。
- B ✅ VFS 向上层提供统一的操作接口（open/read/write 等），屏蔽 ext4/NTFS/FAT32 等底层文件系统的差异。
- C ✅ VFS 的四种核心对象：超级块对象（super_block）、inode 对象（vnode）、目录项对象（dentry）、文件对象（file）。
- **D ❌** VFS 本身不负责数据的实际读写。实际的数据写入由具体文件系统（如 ext4）通过设备驱动程序完成。VFS 仅负责将请求转发给对应的文件系统实现。

---

**11. B（2 次）**

盘块大小 = 4096B，地址大小 = 4B。
直接地址：10 个，覆盖字节范围 0 ~ 10×4096-1 = 0 ~ 40959。
50000 > 40959，因此不在直接地址范围内。

一次间接地址：1 个，每个索引块存放 1024 个地址，覆盖 1024×4096 = 4,194,304B。
一次间接覆盖字节范围：40960 ~ 40960+4194304-1 = 40960 ~ 4235263。

50000 在 40960 ~ 4235263 之间，因此落于一次间接范围内。

访盘次数：
1. 读取一次间址块（索引块），获取数据块地址
2. 读取数据块
共计 **2 次磁盘访问**。

- A ❌ 误算为直接地址范围。
- **B ✅** 正确。
- C ❌ 可能误判为二次间接。
- D ❌ 对应三次间接。

---

**12. D（约 4TB）**

每个指针指向一个盘块（4KB），每个索引块可容纳 4096/4 = 1024 个指针。

| 级别 | 指针数 | 数据量 |
|:----:|:------:|:------:|
| 直接地址（10 个） | 10 个指针 | 10 × 4KB = 40KB |
| 一次间接 | 1024 个指针 | 1024 × 4KB = 4MB |
| 二次间接 | 1024² = 1M 个指针 | 1M × 4KB = 4GB |
| 三次间接 | 1024³ = 1G 个指针 | 1G × 4KB = 4TB |

总大小 ≈ 4TB + 4GB + 4MB + 40KB ≈ **4TB**。

- A ❌ 40KB 仅对应直接地址。
- B ❌ 4MB 对应一次间接。
- C ❌ 4GB 对应二次间接。
- **D ✅** 正确，三次间接提供主要容量。

---

**13. B（8MB）**

- 磁盘容量 = 256GB = 256 × 2³⁰ B = 2³⁸ B
- 盘块大小 = 4KB = 2¹² B
- 总盘块数 = 2³⁸ / 2¹² = 2²⁶ = 67,108,864 个盘块
- 每个盘块需 1 位 → 共需 2²⁶ 位
- 字节数 = 2²⁶ / 2³ = 2²³ B = 8,388,608 B = **8 MB**

- A ❌ 1MB 误用 256GB/256KB 等错误近似。
- **B ✅** 正确。
- C ❌ 64MB 可能误用 256GB/4KB = 64M 块，然后误以为每块 1B。
- D ❌ 128MB 可能误用 256GB/2KB 等错误计算。

---

**14. A（仅 I、III、IV）**

- **I ✅** 直接地址项直接指向文件数据块，其中存放的是文件数据本身。
- **II ❌** 一次间接地址项指向的索引块中存放的是**文件数据块**的盘块号，不是其他地址项的索引信息。二次间接地址项指向的索引块中存放的才是一次间接索引块的块号。
- **III ✅** 若 inode 已在内存中，直接地址可直接获取数据块地址，只需 1 次磁盘 I/O 读取数据块。
- **IV ✅** 三次间接可寻址 1024³ 个盘块 ≈ 4TB，容量最大。

---

**15. A（仅 I、III、IV）**

- **I ✅** inode 中不包含文件名。目录项由"文件名 + inode 编号"组成，文件名存储在目录项中。
- **II ❌** 硬链接指向 inode 编号，而 inode 编号仅在**同一文件系统内**唯一。不同文件系统的 inode 编号空间相互独立，因此硬链接**不能跨文件系统**。软链接存储路径名，可以跨文件系统。
- **III ✅** 软链接存储的是目标文件的路径名。原文件被删除后，通过软链接访问时会因找不到目标文件而失败。
- **IV ✅** 采用 inode 后目录项变小（仅文件名 + inode 号），每个盘块可存放更多目录项，检索时所需读取的盘块数减少，磁盘 I/O 次数相应减少。
